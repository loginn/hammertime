## Cross-Phase Integration

### Connected (14/15 exports wired)

All major cross-phase connections verified:
- `HeroArchetype` class consumed by `hero.gd`, `save_manager.gd`, `main_view.gd`, `forge_view.gd`

- Save format v8 round-trips hero_archetype_id correctly
- Prestige wipe → hero_archetype null → overlay trigger chain works
- Passive bonus injection paths all null-guarded for classless Adventurer

### Missing Connection (1 gap)

**`GameEvents.hero_selected` — emitted but never consumed**

- **Emitted at:** `scenes/main_view.gd:285`
- **Listeners:** None — `grep "hero_selected.connect"` returns 0 results
- **Impact:** After player picks a hero on the overlay, `forge_view.gd` stat panel does not refresh. The hero title and passive bonuses won't appear until the next user-triggered refresh (equip, hover, craft).
- **Fix:** Add `GameEvents.hero_selected.connect(func(_h): update_hero_stats_display())` in `forge_view.gd._ready()`
- **Affected requirements:** PASS-03, SEL-01
