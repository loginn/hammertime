<task type="auto">
  <name>Task 2: Create ForgeView script merging crafting and hero logic</name>
  <files>scenes/forge_view.gd</files>
  <action>
  Create scenes/forge_view.gd extending Node2D. This script consolidates the logic from crafting_view.gd and hero_view.gd into a single unified script.

  **Signals to expose (for MainView to connect):**
  - `signal equipment_changed()` — emitted when player equips an item (for gameplay_view to refresh)

  **Methods to accept from MainView (called by parent):**
  - `func set_new_item_base(item_base: Item) -> void` — from gameplay_view.item_base_found
  - `func on_currencies_found(drops: Dictionary) -> void` — from gameplay_view.currencies_found

  **Merged enum:**
  ```gdscript
  enum ItemSlot { NONE = -1, WEAPON, HELMET, ARMOR, BOOTS, RING }
  ```

  **Node references (@onready):**
  - 6 hammer buttons (RunicHammerBtn through TuningHammerBtn)
  - FinishItemButton
  - ItemImage (TextureRect)
  - 5 item type buttons
  - ItemStatsLabel
  - HeroStatsLabel
  - MeltButton, EquipButton
  - InventoryLabel (optional, for inventory display in sidebar)

  **State variables (from crafting_view.gd):**
  - `var currencies: Dictionary` — instances of all 6 Currency types
  - `var selected_currency: Currency = null`
  - `var selected_currency_type: String = ""`
  - `var currency_buttons: Dictionary = {}`
  - `var current_item: Item` — item currently on crafting bench
  - `var finished_item: Item = null` — item that has been "finished" and awaits melt/equip
  - `var inventory_types = ["weapon", "helmet", "armor", "boots", "ring"]`

  **State variables (from hero_view.gd):**
  - `var currently_hovered_type: String = ""` — tracks which item type button is hovered for hero stats display

  **_ready() logic:**
  1. Initialize currency_buttons mapping (same as crafting_view.gd)
  2. Connect all hammer button signals with `_on_currency_selected.bind(type)`
  3. Connect FinishItemButton
  4. Connect item type buttons with `_on_item_type_selected.bind(type)`
  5. Connect MeltButton.pressed to `_on_melt_pressed`
  6. Connect EquipButton.pressed to `_on_equip_pressed`
  7. Connect ItemImage.gui_input to `update_item` (clicking the item image applies currency)
  8. Connect item type button mouse_entered/mouse_exited for hover behavior
  9. Load crafting inventory from GameState (same pattern as current crafting_view._ready)
  10. Update all displays

  **Crafting logic (from crafting_view.gd, largely unchanged):**
  - `_on_currency_selected(currency_type)` — select/deselect hammer
  - `update_item(event)` — apply currency on click (validate, spend, apply)
  - `update_currency_button_states()` — refresh button text/disabled state
  - `_on_item_type_selected(item_type)` — switch crafting bench item type
  - `update_item_type_button_states()` — highlight active type button
  - `finish_item()` — finish crafting, set finished_item, clear current_item. Do NOT emit item_finished signal or auto-equip. Player must choose Melt or Equip.
  - `add_item_to_inventory(item)` — add item base to crafting inventory
  - `update_inventory_display()` — show inventory in sidebar label

  **NEW: Melt/Equip logic:**
  ```gdscript
  func _on_melt_pressed() -> void:
      if finished_item == null:
          return
      # Destroy the item, free the crafting slot
      print("Melted: ", finished_item.item_name)
      finished_item = null
      update_item_stats_display()
      update_melt_equip_states()

  func _on_equip_pressed() -> void:
      if finished_item == null:
          return
      var slot_name := get_item_type(finished_item)
      if slot_name == "None":
          return
      # Old item in slot is DESTROYED (no swap-back, per CONTEXT.md)
      GameState.hero.equip_item(finished_item, slot_name)
      GameEvents.equipment_changed.emit(slot_name, finished_item)
      print("Equipped: ", finished_item.item_name, " to ", slot_name)
      finished_item = null
      update_hero_stats_display()
      update_item_stats_display()
      update_melt_equip_states()
      equipment_changed.emit()

  func update_melt_equip_states() -> void:
      var has_finished := finished_item != null
      melt_button.disabled = not has_finished
      equip_button.disabled = not has_finished
  ```

  **Hero stats display (from hero_view.gd, adapted):**
  - `update_hero_stats_display()` — builds offense + defense stats text from GameState.hero (same logic as hero_view.update_stats_display). When an item type button is hovered or selected, also shows the equipped item of that type for comparison.
  - `update_item_stats_display()` — shows stats of the finished item (if any) or the current crafting bench item. Reuses get_item_stats_text() from hero_view.gd.

  **Item type hover behavior:**
  When hovering an item type button, the HeroStatsPanel temporarily shows the equipped item of that type (for comparison with the item on the crafting bench). On mouse_exited, revert to standard hero stats.

  ```gdscript
  func _on_type_hover_entered(item_type: String) -> void:
      currently_hovered_type = item_type
      update_hero_stats_display()

  func _on_type_hover_exited(_item_type: String) -> void:
      currently_hovered_type = ""
      update_hero_stats_display()
  ```

  When `currently_hovered_type != ""`, hero_stats_label shows the equipped item stats for that type (using get_item_stats_text). Otherwise shows aggregate hero stats.

  **get_item_stats_text(item: Item) -> String:**
  Copy this method from hero_view.gd. It formats weapon/armor/helmet/boots/ring stats with affixes.

  **update_label() — crafting bench item display:**
  Instead of using a separate Label node for the item name (like the old crafting_view), display the crafting bench item info within the ItemStatsPanel. The ItemStatsLabel shows:
  - If finished_item exists: finished item stats (with "Finished!" header)
  - Else if current_item exists: current crafting bench item stats
  - Else: "No item on crafting bench"

  **Hero stats update timing (per CONTEXT.md):**
  Stats update INSTANTLY on equip. Stats do NOT update during crafting. This is naturally handled because `update_hero_stats_display()` reads from `GameState.hero` which only changes on equip.

  **Important:** Do NOT keep any references to old hero_view equipment slot Buttons (WeaponSlot, HelmetSlot, etc.). Equipping is now done via the Equip button, not by clicking equipment slots.
