# 41-01: End-to-end prestige loop verification test scene

## Goal
Create a GDScript test scene that verifies the full prestige loop (P0 to P1), save round-trips, item tier gating, tag hammer gating, and crafting regression -- confirming all 23 v1.7 requirements work together correctly.

## must_haves
- [ ] Test scene runs as standalone scene from Godot editor with structured [PASS]/[FAIL] output and summary
- [ ] Full prestige flow verified: fresh game -> grant 100 forge hammers -> execute prestige -> post-prestige state correct (prestige_level=1, max_item_tier_unlocked=7, area/hero/inventory reset, 1 tag currency granted)
- [ ] Save round-trip at P0 and P1 verified via _build_save_data()/_restore_state() with exact field comparison
- [ ] Item tier -> affix tier floor gating verified (tier 7 item produces affixes >= tier 25, tier 8 produces >= 29)
- [ ] Crafting regression verified: RunicHammer applies to starter weapon after prestige, adds at least 1 mod
- [ ] Tag hammer gating verified: prestige_level < 1 blocks tag section, prestige_level >= 1 enables it; TagHammer.can_apply() requires Normal rarity
- [ ] File I/O round-trip verified: write save data to user://test_save.json, read back, restore, compare -- cleanup after
- [ ] No real save file overwritten (avoids calling save_game()/execute_prestige() directly; uses manual prestige steps or disconnects prestige_completed signal)

<task id="41-01-02">
### Create test script (integration_test.gd)

Create `tools/test/integration_test.gd` extending Node. All tests run in `_ready()` after autoloads have initialized. The script must:

**Test infrastructure:**
- Track pass/fail counts with `_pass_count` and `_fail_count` int vars
- Helper `_check(condition: bool, description: String)` that prints `[PASS] description` or `[FAIL] description` and increments counters
- Helper `_reset_fresh()` that calls `GameState.initialize_fresh_game()` to get clean baseline before each group
- Helper `_simulate_prestige()` that manually replicates prestige steps WITHOUT calling `execute_prestige()` (to avoid prestige_completed signal triggering auto-save):
  1. Spend forge currency (loop 100 times calling `GameState.spend_currency("forge")`)
  2. Set `GameState.prestige_level = 1`
  3. Set `GameState.max_item_tier_unlocked = PrestigeManager.ITEM_TIERS_BY_PRESTIGE[1]` (= 7)
  4. Call `GameState._wipe_run_state()`
  5. Grant 1 tag currency: `GameState.tag_currency_counts["fire"] = 1` (deterministic, not random)
- Print `\n=== GROUP NAME ===` header before each group
- Print `\n=== SUMMARY ===\nX/Y tests passed` at end
- If any tests fail, print `[FAILURES DETECTED]`; otherwise print `[ALL PASSED]`

**Group 1: Pre-Prestige Baseline (P0)** -- verifies PRES-03, PRES-05
1. `_reset_fresh()`
2. Check `prestige_level == 0`
3. Check `max_item_tier_unlocked == 8`
4. Check `can_prestige() == false` (no forge hammers)
5. Check starter weapon exists: `crafting_inventory["weapon"].size() >= 1` and first item `is LightSword`
6. Check `area_level == 1`
7. Check `tag_currency_counts.is_empty()`

**Group 2: Prestige Gating** -- verifies PRES-01
1. Grant 99 forge hammers (`GameState.currency_counts["forge"] = 99`)
2. Check `can_prestige() == false`
3. Grant 1 more (`GameState.currency_counts["forge"] = 100`)
4. Check `can_prestige() == true`
5. Check `PrestigeManager.get_next_prestige_cost() == {"forge": 100}`

**Group 3: Execute Prestige P0 -> P1** -- verifies PRES-01, PRES-02, PRES-03, PRES-06, TAG-07, TAG-08
1. `_reset_fresh()`, grant 100 forge hammers
2. Call `_simulate_prestige()`
3. Check `prestige_level == 1`
4. Check `max_item_tier_unlocked == 7`
5. Check `area_level == 1` (reset)
6. Check all hero equipment slots are null (iterate `hero.equipped_items`, check each `== null`)
7. Check starter weapon in crafting inventory (`crafting_inventory["weapon"].size() >= 1`, first `is LightSword`)
8. Check `currency_counts["forge"] == 0` (spent + wiped)
9. Check `currency_counts["runic"] == 1` (fresh-game default after wipe)
10. Check tag_currency_counts total == 1 (sum all values)
11. Check `can_prestige() == false` (P2 costs 999999)

**Group 4: Save Round-Trip at P0** -- verifies SAVE-01
1. `_reset_fresh()`
2. Set known state: `prestige_level = 0`, `max_item_tier_unlocked = 8`, `tag_currency_counts = {}`
3. Call `var saved := SaveManager._build_save_data()`
4. Trash state: `GameState.prestige_level = 99`, `GameState.max_item_tier_unlocked = 1`, `GameState.tag_currency_counts = {"fire": 999}`
5. Call `SaveManager._restore_state(saved)`
6. Check `prestige_level == 0`
7. Check `max_item_tier_unlocked == 8`
8. Check `tag_currency_counts.is_empty()` or size == 0
9. Check `area_level == 1`
10. Check `currency_counts["runic"] == 1`

**Group 5: Save Round-Trip at P1** -- verifies SAVE-01
1. `_reset_fresh()`, grant 100 forge, call `_simulate_prestige()`
2. Set `tag_currency_counts = {"fire": 1, "cold": 2}` (known values)
3. Call `var saved := SaveManager._build_save_data()`
4. Trash state: `GameState.prestige_level = 0`, `GameState.max_item_tier_unlocked = 8`, `GameState.tag_currency_counts = {}`
5. Call `SaveManager._restore_state(saved)`
6. Check `prestige_level == 1`
7. Check `max_item_tier_unlocked == 7`
8. Check `tag_currency_counts.get("fire", 0) == 1`
9. Check `tag_currency_counts.get("cold", 0) == 2`
10. Check hero exists and is functional (`GameState.hero != null`)

**Group 6: Crafting Regression After Prestige** -- verifies AFFIX-01, AFFIX-02
1. `_reset_fresh()`, set `prestige_level = 1`
2. Get starter weapon: `var weapon: Item = GameState.crafting_inventory["weapon"][0]`
3. Check `weapon.rarity == Item.Rarity.NORMAL`
4. Apply RunicHammer: `var hammer := RunicHammer.new(); hammer.apply(weapon)`
5. Check `weapon.rarity == Item.Rarity.MAGIC`
6. Check `weapon.prefixes.size() + weapon.suffixes.size() >= 1` (at least 1 mod added)

**Group 7: Item Tier / Affix Tier Floor** -- verifies TIER-01, TIER-03
1. Create `var sword := LightSword.new()` (tier 8 by default)
2. Check `sword._get_affix_tier_floor() == 29`
3. Set `sword.tier = 7`
4. Check `sword._get_affix_tier_floor() == 25`
5. Set `sword.tier = 1`
6. Check `sword._get_affix_tier_floor() == 1`
7. Test affix generation respects floor: create fresh LightSword (tier 7), set rarity NORMAL, apply RunicHammer
8. For each affix in `prefixes + suffixes`, check `affix.tier >= 25` (tier 7 floor)

**Group 8: Tag Hammer Gating (Logic-Only)** -- verifies TAG-01 through TAG-07, PUI-04
1. `_reset_fresh()` (prestige_level = 0)
2. Check `GameState.prestige_level >= 1` is `false` (tag section visibility condition)
3. Set `GameState.prestige_level = 1`
4. Check `GameState.prestige_level >= 1` is `true`
5. Create `var fire_hammer := TagHammer.new("FIRE", "Fire Hammer")`
6. Create fresh LightSword (Normal rarity)
7. Check `fire_hammer.can_apply(sword) == true` (LightSword has no FIRE valid_tag but some affixes have FIRE tag -- verify result; if false, use "PHYSICAL" tag instead since LightSword has PHYSICAL valid_tag)
8. Set sword rarity to MAGIC, check `fire_hammer.can_apply(sword) == false` (requires Normal)
9. Test tag currency spend: set `GameState.tag_currency_counts["fire"] = 1`
10. Check `GameState.spend_tag_currency("fire") == true`
11. Check `GameState.tag_currency_counts["fire"] == 0`
12. Check `GameState.spend_tag_currency("fire") == false` (none left)

**Group 9: File I/O Round-Trip** -- verifies SAVE-01, SAVE-02
1. `_reset_fresh()`, set known prestige state (`prestige_level = 1, max_item_tier_unlocked = 7, tag_currency_counts = {"cold": 3}`)
2. Call `var data := SaveManager._build_save_data()`
3. Write to `user://test_save.json` via `FileAccess.open("user://test_save.json", FileAccess.WRITE)`, store `JSON.stringify(data)`
4. Read back: `FileAccess.open("user://test_save.json", FileAccess.READ)`, `JSON.parse_string(text)`
5. Check parsed data is Dictionary
6. Trash state, call `SaveManager._restore_state(parsed)`
7. Check `prestige_level == 1`, `max_item_tier_unlocked == 7`, `tag_currency_counts.get("cold", 0) == 3`
8. Clean up: `DirAccess.remove_absolute("user://test_save.json")`
9. Check file no longer exists: `FileAccess.file_exists("user://test_save.json") == false`

**Important implementation notes:**
- The `apply()` method on Currency is the public API (calls `_do_apply()` internally). Check if `apply()` exists; if not, call `_do_apply()` directly.
- `TagHammer._init()` takes `(p_tag, p_name)` -- tag must be uppercase string matching Tag constants (e.g., `"FIRE"`, `"PHYSICAL"`).
- LightSword `valid_tags` are `[Tag.PHYSICAL, Tag.ATTACK, Tag.CRITICAL, Tag.WEAPON]`. For TagHammer testing, use `"PHYSICAL"` tag which matches LightSword's valid_tags. Fire-tagged affixes may or may not match LightSword depending on their tags -- check `can_apply()` result and adapt.
- All affix `tier_range` values are `Vector2i(1, 32)`. The `affix.tier` field holds the rolled tier after construction.
- After `_restore_state()`, call assertions immediately (state is already restored).
