**2. Add `_on_stash_slot_pressed` function** (per D-05, D-06, D-07, D-08):

```gdscript
func _on_stash_slot_pressed(slot_type: String, index: int) -> void:
    # D-05: bench must be empty to load from stash
    if GameState.crafting_bench != null:
        _show_forge_error("Melt or equip first")
        return

    var items: Array = GameState.stash.get(slot_type, [])
    if index >= items.size() or items[index] == null:
        return  # Empty slot guard

    # Transfer item from stash to bench
    var item: Item = items[index]
    # D-08: remove at index, leave gap (no shifting — remove_at shifts remaining, so set null instead)
    items[index] = null
    GameState.crafting_bench = item
    current_item = item

    # D-07: flash the now-empty slot
    _flash_stash_slot(slot_type, index)

    # Refresh all displays
    update_current_item()
    _update_stash_display()
    update_melt_equip_states()
    update_inventory_display()
```

**IMPORTANT per D-08:** The research code used `remove_at(index)` but D-08 says "remaining items do not shift to fill it." Using `remove_at` WOULD shift items. Instead, set `items[index] = null` to leave the gap. Then `_update_stash_display()` must treat null entries as empty (which it already does: `if i < items.size() and items[i] != null`).

BUT there is a complication: `GameState.stash["weapon"]` arrays start as `[]` and grow via `add_item_to_stash()` which uses `append()`. So arrays may be size 0, 1, 2, or 3. Setting `items[index] = null` works when `index < items.size()`. The null stays in the array, preserving positions. This is correct for D-08.

    # Stash full for this slot
