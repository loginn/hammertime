<task type="auto">
  <name>Task 2: Implement state transition visual feedback with delays</name>
  <files>scenes/gameplay_view.gd</files>
  <action>
Add visual state transition handling to gameplay_view.gd signal handlers:

1. **Pack-to-pack pause (~0.5 second):**
   The CombatEngine currently transitions instantly between packs (_on_pack_killed starts next fight immediately). To add visual pause without modifying CombatEngine's logic, add a brief delay in CombatEngine._on_pack_killed() between killing a pack and starting the next fight.

   Actually — modify combat_engine.gd to add a pack_transition_delay_sec: float = 0.5 property. In _on_pack_killed(), after recharging ES and before _start_pack_fight(), use:
   ```
   await get_tree().create_timer(pack_transition_delay_sec).timeout
   ```
   This creates the user-requested "brief visual pause so player notices pack change."

2. **Death delay (2-3 seconds):**
   Modify combat_engine.gd _on_hero_died(). Before auto-retry start_combat, add:
   ```
   await get_tree().create_timer(2.5).timeout
   ```
   This creates "auto-retry after a short delay (2-3 seconds), no button needed."

3. **Death inline state change:**
   In gameplay_view.gd _on_hero_died(), update combat_state_label.text to "Hero died! Retrying..." (if auto_retry) or "Hero died! Click Start Combat to retry." Update the label color to a warning color (e.g., Color(1.0, 0.3, 0.3)) during death, then reset when combat restarts.

4. **Map complete seamless advance:**
   In gameplay_view.gd _on_map_completed(), briefly flash combat_state_label with "Map Clear!" text (the auto-advance in CombatEngine already handles progression). Reset label when new combat starts.

5. **Combat state label updates in each handler:**
   - _on_combat_started: "Fighting..." in normal color
   - _on_pack_killed: "Pack cleared!" briefly (will reset on next combat_started or pack fight)
   - _on_hero_died: "Hero died! Retrying..." in warning color
   - _on_map_completed: "Map Clear!" in success color (e.g., Color(0.2, 0.9, 0.2))
   - _on_combat_stopped: "Combat stopped." in normal color

NOTE: files_modified for this task includes models/combat/combat_engine.gd for the delay additions. Update the plan's files_modified frontmatter accordingly.
