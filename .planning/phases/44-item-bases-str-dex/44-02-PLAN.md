<task id="44-02-02">
<title>Rewrite get_random_item_base() with slot-first-then-archetype drop logic</title>
<details>
Update `scenes/gameplay_view.gd` function `get_random_item_base()` (around line 274).

Replace the current implementation with slot-first-then-archetype logic:

```gdscript
func get_random_item_base() -> Item:
    # Roll tier from area-weighted distribution
    var tier = LootTable.roll_item_tier(GameState.area_level, GameState.max_item_tier_unlocked)

    # Slot-first: pick random slot (20% each)
    var slots = ["weapon", "armor", "helmet", "boots", "ring"]
    var slot = slots[randi() % slots.size()]

    # Then archetype: pick random base within slot
    var bases: Dictionary = {
        "weapon": [Broadsword, Battleaxe, Warhammer, Dagger, VenomBlade, Shortbow],
        "armor": [IronPlate, LeatherVest, SilkRobe],
        "helmet": [IronHelm, LeatherHood, Circlet],
        "boots": [IronGreaves, LeatherBoots, SilkSlippers],
        "ring": [IronBand, JadeRing, SapphireRing],
    }

    var slot_bases = bases[slot]
    var base_class = slot_bases[randi() % slot_bases.size()]
    # Items always drop as Normal (0 affixes) -- crafting is the sole source of mods
    return base_class.new(tier)
```

Key changes:
1. Tier rolled BEFORE construction (passed to constructor)
2. 5 slots with equal weight (20% each)
3. Within each slot, equal weight per base type
4. Weapons have 6 bases (3 STR + 3 DEX) — effectively 50/50 STR/DEX split
5. Non-weapon slots have 3 bases (1 STR + 1 DEX + 1 INT) — 33% each archetype

Remove the old `item_types` array and post-construction tier assignment.
</details>
