<task id="44-03-02">
<title>Update integration tests for all 21 item types</title>
<details>
Update `tools/test/integration_test.gd` with the following changes:

**1. Replace all existing LightSword references** with Broadsword:
- Search for `LightSword` in the file and replace with `Broadsword`
- Update any `LightSword.new()` calls to `Broadsword.new(8)` (or appropriate tier)
- Update any `is LightSword` checks to `is Broadsword`
- Update any string comparisons like `"LightSword"` to `"Broadsword"`

**2. Replace all BasicArmor/BasicHelmet/BasicBoots/BasicRing references** similarly:
- `BasicArmor` -> `IronPlate`
- `BasicHelmet` -> `IronHelm`
- `BasicBoots` -> `IronGreaves`
- `BasicRing` -> `IronBand`
- All `.new()` calls should pass tier parameter: `.new(8)`

**3. Add new test groups** following the existing test group pattern in the file:

**Group 10: Item Base Construction (all 21 types)**
For each of the 21 item base types, at tiers 8 and 1:
- Construct the item
- Verify item_name matches expected tier name (non-empty string)
- Verify get_item_type_string() returns the correct type string
- Verify tier matches the constructed tier
- Verify valid_tags is non-empty and contains the expected archetype tag (STR, DEX, or INT)
- For weapons: verify base_damage_min > 0 and base_damage_max > base_damage_min
- For defense items: verify primary defense stat > 0
- For rings: verify base_damage > 0
- Verify T1 stats > T8 stats (scaling works)

Use a helper function to avoid repetitive code:
```gdscript
func _test_item_construction(item_class, type_string: String, archetype_tag: String, tier: int) -> bool:
    var item = item_class.new(tier)
    if item.get_item_type_string() != type_string: return false
    if item.tier != tier: return false
    if item.item_name == "": return false
    if not item.valid_tags.has(archetype_tag): return false
    return true
```

**Group 11: Serialization Round-Trip (all 21 types)**
For each of the 21 types at tier 5:
- Construct the item
- Call to_dict()
- Call Item.create_from_dict(dict)
- Verify restored item_name matches
- Verify restored tier matches
- Verify get_item_type_string() matches

**Group 12: Defense Archetype Verification**
- Create IronPlate(5), LeatherVest(5), SilkRobe(5)
- Verify IronPlate: computed_armor > 0, computed_evasion == 0, computed_energy_shield == 0
- Verify LeatherVest: computed_evasion > 0, computed_armor == 0
- Verify SilkRobe: computed_energy_shield > 0, computed_armor == 0
- Verify total_defense reflects primary stat for each

**Group 13: Valid Tags / Affix Gating**
- Create Broadsword(8) — verify has_valid_tag returns true for a Physical Damage prefix (tags include PHYSICAL + WEAPON)
- Create Dagger(8) — verify has_valid_tag returns true for a Crit suffix (tags include CRITICAL)
- Create IronPlate(8) — verify has_valid_tag returns true for Flat Armor affix (has ARMOR) and false for an Evasion-only affix
- Create LeatherVest(8) — verify has_valid_tag returns true for Evasion affix (has EVASION) and false for Armor-only affix

**Group 14: Drop Generation**
- Call gameplay_view.get_random_item_base() 100+ times (or simulate the logic inline if calling the function is impractical from the test)
- Verify all 5 slot types represented (check `is Weapon`, `is Armor`, `is Helmet`, `is Boots`, `is Ring`)
- Verify items have valid tier (1-8) and non-empty item_name

**Group 15: Starter Weapon**
- Verify GameState.crafting_inventory["weapon"] after initialize_fresh_game() is a Broadsword
- Verify tier == 8
- Verify item_name == "Rusty Broadsword"
