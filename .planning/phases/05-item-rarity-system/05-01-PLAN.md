<task type="auto">
  <name>Task 1: Add Rarity enum, limits mapping, and rarity property to Item</name>
  <files>models/items/item.gd</files>
  <action>
Add the following to the Item class (models/items/item.gd):

1. **Rarity enum** at the top of the class (after class_name):
   ```
   enum Rarity { NORMAL, MAGIC, RARE }
   ```

2. **Configurable rarity-to-limits mapping** as a class-level const dictionary. Per user decision, this must NOT be a hardcoded match statement:
   ```
   const RARITY_LIMITS: Dictionary = {
       Rarity.NORMAL: { "prefixes": 0, "suffixes": 0 },
       Rarity.MAGIC: { "prefixes": 1, "suffixes": 1 },
       Rarity.RARE: { "prefixes": 3, "suffixes": 3 },
   }
   ```

3. **Rarity property** with default Normal:
   ```
   var rarity: Rarity = Rarity.NORMAL
   ```

4. **Custom override properties** (null = use rarity defaults). Per user decision, base types CAN override limits for future exotic bases. For v1.0 no base types use these, but the mechanism must exist:
   ```
   var custom_max_prefixes = null
   var custom_max_suffixes = null
   ```

5. **Computed max_prefixes() and max_suffixes() functions** that check custom overrides first, then fall back to RARITY_LIMITS:
   ```
   func max_prefixes() -> int:
       if custom_max_prefixes != null:
           return custom_max_prefixes
       return RARITY_LIMITS[rarity]["prefixes"]

   func max_suffixes() -> int:
       if custom_max_suffixes != null:
           return custom_max_suffixes
       return RARITY_LIMITS[rarity]["suffixes"]
   ```

6. **Update add_prefix()**: Change the hardcoded `>= 3` check to use `max_prefixes()`. Return `false` when limit is reached (instead of just printing). Return `true` on success. This replaces the existing add_prefix() function entirely. Keep the existing logic for finding valid prefixes via tags and duplicate checking, but use `max_prefixes()` for the limit:
   ```
   func add_prefix() -> bool:
       if len(self.prefixes) >= max_prefixes():
           print("Cannot add more prefixes - at rarity limit (%d)" % max_prefixes())
           return false
       # ... rest of existing valid prefix selection logic ...
       return true  # after successful add
   ```
   Note: The `return false` early-exit also covers Normal items (max_prefixes() returns 0, and `0 >= 0` is true).

7. **Update add_suffix()**: Same pattern as add_prefix() — change `>= 3` to `max_suffixes()`, return bool.

The existing `is_affix_on_item()`, `has_valid_tag()`, `reroll_affix()` functions stay unchanged.
