  <action>
Create `models/currencies/tag_hammer.gd` as a single parameterized class for all 5 tag hammers.

Class structure: `class_name TagHammer extends Currency`

Constructor: `func _init(p_tag: String, p_name: String) -> void` — stores `required_tag` and sets `currency_name`.

**can_apply():** Returns false if `item.rarity != Item.Rarity.NORMAL`. Also returns false if `_has_any_matching_affix(item)` is false. Both conditions checked.

**get_error_message():** Returns "Fire Hammer can only be used on Normal items" (using `currency_name`) if rarity check fails. Returns "No {tag}-tagged mods available for this item" (lowercase `required_tag`) if no matching affix available. Uses `required_tag.to_lower()` in the message.

**_do_apply():**
1. Set `item.rarity = Item.Rarity.RARE`
2. Roll 4-6 mods using exact same pattern as ForgeHammer (50/50 prefix/suffix, fallback to other type on failure)
3. After rolling, check `_has_matching_affix_on_item(item)` — if false, call `_replace_random_affix_with_tagged(item)`
4. Call `item.update_value()`

**_has_any_matching_affix(item):** Check whether at least one valid affix pool entry exists for this item with the required tag. Loop over `ItemAffixes.prefixes` and `ItemAffixes.suffixes`. For each template, check BOTH `item.has_valid_tag(template)` AND `required_tag in template.tags`. Return true on first match. This is the pre-application check.

**_has_matching_affix_on_item(item):** Loop `item.prefixes` and `item.suffixes`. Return true if any affix has `required_tag in affix.tags`. Post-application check.

**_replace_random_affix_with_tagged(item):**
1. Get floor: `var floor_val := item._get_affix_tier_floor()`
2. Build tagged prefix pool: templates from `ItemAffixes.prefixes` where `required_tag in template.tags` AND `item.has_valid_tag(template)`. Skip `is_affix_on_item` dedup here — we're replacing, not adding, and the pool may be tiny.
3. Build tagged suffix pool: same from `ItemAffixes.suffixes`
4. If both pools empty, return (should not happen — can_apply guards this)
5. Try to replace a prefix victim with a prefix replacement (if prefix pool not empty and item has prefixes): pick random prefix victim, pick random template from prefix pool, create replacement via `Affixes.from_affix(template, floor_val)`, replace at the same index.
6. If prefix pool is empty or item has no prefixes, fall back: replace a suffix victim with a suffix replacement.
7. If suffix pool is also empty: pick any victim (prefix or suffix), try prefix pool first then suffix pool, replace regardless of type mismatch (last-resort fallback only).
8. The goal: always end up with at least one tagged affix on the item.

**CRITICAL pitfall to avoid:** Do NOT mix prefix/suffix types in replacement. Research notes this as the primary pitfall — prefix victim must get prefix replacement, suffix victim must get suffix replacement. Only fall back to cross-type replacement if no same-type option exists.

**CRITICAL pitfall to avoid:** Do NOT route tag currency spending through `GameState.spend_currency()` — that only reads `currency_counts`. Tag currencies use `GameState.spend_tag_currency()` (added in Task 2).
