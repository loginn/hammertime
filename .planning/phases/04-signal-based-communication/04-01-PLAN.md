<objective>
Replace all sibling get_node() calls with signal-based communication following Godot's "call down, signal up" pattern.

Purpose: Eliminate brittle cross-view coupling where views find siblings via get_node("../SiblingView"). Instead, child views emit typed signals and the parent (main_view) coordinates by connecting those signals to the appropriate sibling methods. This makes the scene structure refactor-safe.

Output: Four refactored view scripts where all cross-view communication flows through signals, with main_view.gd acting as the parent coordinator.
</objective>

Refactor main_view.gd to be the parent coordinator that connects child signals to sibling methods, using @onready caching for all node references.

1. Replace the plain `var` declarations (lines 4-6) with @onready vars:
```gdscript
@onready var crafting_view: Node2D = $CraftingView
@onready var hero_view: Node2D = $HeroView
@onready var gameplay_view: Node2D = $GameplayView
@onready var crafting_button: Button = $NavigationPanel/CraftingButton
@onready var hero_button: Button = $NavigationPanel/HeroButton
@onready var gameplay_button: Button = $NavigationPanel/GameplayButton
```

2. Rewrite `_ready()` to:
   a. Remove the manual `crafting_view = $CraftingView` assignments (lines 13-15) since @onready handles this.
   b. Remove all the `has_node()` guard checks and debug prints -- these were scaffolding from early development. Use the @onready references directly.
   c. Connect navigation buttons using @onready references:
      ```gdscript
      crafting_button.pressed.connect(_on_crafting_button_pressed)
      hero_button.pressed.connect(_on_hero_button_pressed)
      gameplay_button.pressed.connect(_on_gameplay_button_pressed)
      ```
   d. Add signal coordination (the core of this task):
      ```gdscript

      crafting_view.item_finished.connect(hero_view.set_last_crafted_item)

      gameplay_view.item_base_found.connect(crafting_view.set_new_item_base)
      gameplay_view.hammers_found.connect(crafting_view.add_hammers)
      ```
   e. Keep `show_view("crafting")` call.

3. Rewrite `show_view()` to use @onready cached button references (`crafting_button`, `hero_button`, `gameplay_button`) instead of `$NavigationPanel/CraftingButton` etc.

4. Remove the debug `print()` statements throughout `_ready()` -- they were development scaffolding. Keep the `print()` in `show_view()` at the bottom since that is runtime feedback.

5. Remove the `if has_node("NavigationPanel"): $NavigationPanel.modulate = Color.WHITE` block -- unnecessary defensive code.
