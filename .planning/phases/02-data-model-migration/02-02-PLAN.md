<objective>
Create GameState and GameEvents autoloads, convert Hero to Resource, and wire views to use GameState.hero instead of local hero instances.

Purpose: GameState provides a single source of truth for the Hero instance -- no more views creating their own Hero or reaching into sibling views to share one. GameEvents provides a decoupled event bus for cross-scene signals (equipment changes, crafting, area clearing) that will replace the current get_node("../SiblingView") wiring in Phase 4. Hero extends Resource because it is pure data (stats, equipment) with no scene tree behavior.

Output: 2 new autoload files, 1 modified model (Hero), 3 modified files (project.godot, hero_view.gd, gameplay_view.gd), game runs identically with centralized state.
</objective>

  <action>
**Create autoloads/game_events.gd:**
```gdscript
extends Node

# Core gameplay signals for cross-scene communication
signal equipment_changed(slot: String, item: Item)
signal item_crafted(item: Item)
signal area_cleared(area_level: int)
```
No class_name needed -- autoloads are accessed by their registered name (GameEvents).

**Create autoloads/game_state.gd:**
```gdscript
extends Node

var hero: Hero

func _ready() -> void:
    hero = Hero.new()
    # Initialize empty equipment slots
    hero.equipped_items["weapon"] = null
    hero.equipped_items["helmet"] = null
    hero.equipped_items["armor"] = null
    hero.equipped_items["boots"] = null
    hero.equipped_items["ring"] = null
```
No class_name needed. The equipment slot initialization moves here from hero_view.gd's _ready() -- GameState owns the Hero and its initial state.

**Modify models/hero.gd:**
1. Change `class_name Hero extends Node` to `class_name Hero extends Resource`
2. Everything else stays the same. Hero has no _ready(), no _process(), no scene tree calls. All methods (take_damage, heal, die, revive, equip_item, unequip_item, update_stats, calculate_dps, calculate_defense, calculate_crit_stats, getters) work identically on Resources.

**Modify project.godot:**
