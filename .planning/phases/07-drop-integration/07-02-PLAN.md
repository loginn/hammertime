**Extend LootTable** (`models/loot/loot_table.gd`) with currency drop logic:

Add a static method `roll_currency_drops(area_level: int) -> Dictionary` that returns a dictionary mapping currency name strings to drop counts.

Currency names (keys): `"runic"`, `"forge"`, `"tack"`, `"grand"`, `"claw"`, `"tuning"`

Drop logic:
- Base drop count per currency: each currency has an independent chance to drop per area clear
- For each currency type, roll whether it drops and how many:

```
Per clear, for each currency:
  runic:   70% chance to drop, 1-2 quantity
  forge:   30% chance to drop, 1 quantity
  tack:    50% chance to drop, 1-2 quantity
  grand:   20% chance to drop, 1 quantity
  claw:    40% chance to drop, 1-2 quantity
  tuning:  40% chance to drop, 1-2 quantity
```

- Area level bonus: `area_level - 1` bonus drops distributed randomly across currency types that already dropped (simulates more loot in harder areas). For each bonus drop, pick a random key from the already-dropped currencies and add 1.
- If no currencies dropped at all (unlikely but possible), guarantee 1 runic hammer (most basic currency).
- Return the dictionary with only non-zero entries.

Implementation: Use `randf() < chance` for each currency's drop check. Use `randi_range(min, max)` for quantity. Iterate `area_level - 1` times for bonus drops.

- Otherwise decrements by 1 and returns `true`

All functions need explicit return type hints.
