  <name>Task 1: Create LootTable Resource with rarity weights and item spawning</name>
  <files>models/loot/loot_table.gd</files>
  <action>
Create `models/loot/loot_table.gd` as a Resource class (`class_name LootTable extends Resource`).

**Rarity weight system:**

Add a static method `get_rarity_weights(area_level: int) -> Dictionary` that returns weights for each rarity tier based on area level:

```
Area 1 (Forest):       { NORMAL: 80, MAGIC: 18, RARE: 2 }
Area 2 (Dark Forest):  { NORMAL: 50, MAGIC: 40, RARE: 10 }
Area 3 (Cursed Woods): { NORMAL: 20, MAGIC: 45, RARE: 35 }
Area 4 (Shadow Realm): { NORMAL: 5,  MAGIC: 30, RARE: 65 }
Area 5+ (beyond):      { NORMAL: 2,  MAGIC: 28, RARE: 70 }
```

Store these as a const Dictionary mapping area_level (int) to weight dictionary. For area_level > 4, use the level-5+ weights.

**Rarity rolling:**

Add a static method `roll_rarity(area_level: int) -> Item.Rarity` that:
1. Gets weights from `get_rarity_weights(area_level)`
2. Calculates total weight (sum of all values)
3. Generates `randi_range(1, total_weight)`
4. Walks through the weights accumulating, returning the rarity when accumulated weight >= roll
5. Order: NORMAL first, then MAGIC, then RARE (so lowest roll = NORMAL)

**Item spawning with mods:**

Add a static method `spawn_item_with_mods(item: Item, rarity: Item.Rarity) -> void` that:
- If `rarity == NORMAL`: do nothing (item stays Normal with 0 explicit mods)
- If `rarity == MAGIC`:
  - Set `item.rarity = Item.Rarity.MAGIC`
  - Add 1-2 random mods using the same 50/50 prefix/suffix logic as RunicHammer._do_apply():
    - `randi_range(1, 2)` for mod count
    - For each mod: `randi_range(0, 1) == 0` to choose prefix, try alternate if first fails
  - Call `item.update_value()`
- If `rarity == RARE`:
  - Set `item.rarity = Item.Rarity.RARE`
  - Add 4-6 random mods using same logic as ForgeHammer._do_apply():
    - `randi_range(4, 6)` for mod count
    - For each mod: 50/50 prefix/suffix, break if both fail
  - Call `item.update_value()`

This duplicates the mod-addition logic from RunicHammer/ForgeHammer intentionally -- LootTable spawns items at a rarity, it does NOT consume a currency. The hammers are for player-initiated crafting; LootTable is for drop generation.

All methods must have explicit return type hints per project convention.
