<task type="auto">
  <name>Task 1: Create PackGenerator with scaling and element selection</name>
  <files>models/monsters/pack_generator.gd</files>
  <action>
Create `models/monsters/pack_generator.gd`:
- `class_name PackGenerator extends RefCounted` (follows StatCalculator/DefenseCalculator pattern)

Constants:
- `const PACK_COUNT_MIN: int = 8`
- `const PACK_COUNT_MAX: int = 15`
- `const GROWTH_RATE: float = 0.06` -- 6% exponential growth per level (within user's 5-8% range, chosen as sweet spot per research)

Static methods:

**`static func get_level_multiplier(area_level: int) -> float`**
- Compound exponential: `pow(1.0 + GROWTH_RATE, area_level - 1)`
- At level 1: multiplier = 1.0 (no scaling)
- At level 100: ~321x
- At level 300: ~42,012x

**`static func roll_element(biome: BiomeConfig) -> String`**
- Weighted random selection from biome.element_weights
- Follow LootTable.roll_rarity() pattern: sum weights, roll random, accumulate and compare
- Normalize weights at selection time (divide each by sum) for robustness
- Returns one of "physical", "fire", "cold", "lightning"

**`static func create_pack(monster_type: MonsterType, area_level: int, element: String) -> MonsterPack`**
- Get multiplier from get_level_multiplier(area_level)
- Create MonsterPack with:
  - pack_name = monster_type.type_name
  - hp = monster_type.base_hp * multiplier
  - max_hp = hp (same value at creation)
  - damage = monster_type.base_damage * multiplier
  - attack_speed = monster_type.base_attack_speed (NOT scaled -- preserves type identity per CONTEXT.md)
  - element = element parameter

**`static func generate_packs(area_level: int) -> Array[MonsterPack]`**
- Get biome via BiomeConfig.get_biome_for_level(area_level)
- Roll pack count: randi_range(PACK_COUNT_MIN, PACK_COUNT_MAX)
- For each pack:
  - Pick random monster type from biome.monster_types using pick_random()
  - Roll element from biome using roll_element()
  - Create pack using create_pack()
- Return array of MonsterPack instances
- All packs use the SAME area_level multiplier (no escalation within map, per CONTEXT.md)

All functions must have explicit return type hints.
