**After (new):**
```gdscript
## Hero attacks the current pack. Rolls each damage element independently, then applies crit.
func _on_hero_attack() -> void:
	if state != State.FIGHTING:
		return

	var pack := get_current_pack()
	if pack == null or not pack.is_alive():
		return

	var hero := GameState.hero

	# Roll each element independently from hero's cached damage ranges
	var damage_per_hit := 0.0
	for element in hero.damage_ranges:
		var el_min: float = hero.damage_ranges[element]["min"]
		var el_max: float = hero.damage_ranges[element]["max"]
		if el_max > 0.0:
			damage_per_hit += randf_range(el_min, el_max)

	# Per-hit crit roll (applied to total, not per-element)
	var is_crit := randf() < (hero.total_crit_chance / 100.0)
	if is_crit:
		damage_per_hit *= (hero.total_crit_damage / 100.0)

	pack.take_damage(damage_per_hit)
	GameEvents.hero_attacked.emit(damage_per_hit, is_crit)

	if not pack.is_alive():
		_on_pack_killed()
```

Key changes:
- Replaces `hero.total_dps / hero_attack_speed` with per-element rolling from `hero.damage_ranges`
- Each element with a non-zero max is rolled independently via `randf_range(el_min, el_max)`
- Elements with max=0.0 are skipped (no contribution)
- All element rolls are summed into `damage_per_hit`
- Crit is applied to the total sum (same as before -- one crit roll per attack)
- The `hero_attacked` signal emission and `pack.take_damage()` call are unchanged
- The guard checks (state, pack alive) are unchanged

Note: The old formula divided DPS by attack_speed to remove the speed factor. The new formula doesn't need this because the damage_ranges already represent per-hit damage (they don't include speed). Speed only affects hit frequency (timer cadence), not hit magnitude. The hero.total_dps display value still uses the average-of-ranges * speed * crit formula (from Phase 24), but per-hit combat uses raw rolled values.

Update the method comment to reflect the new behavior.
