  **Step B: Change ramp_duration from 50 to 12**

  In `_calculate_currency_chance()`, update the default `ramp_duration` parameter from 50 to 12. This means newly unlocked currencies reach full drop rate ~12 levels after their unlock threshold (per user decision). Keep the same "immediate but low" shape: starts at a low multiplier, ramps to 1.0 over the duration.

  Use a square-root curve instead of linear for the "immediate but low" feel — this front-loads the early ramp so players see the currency quickly, then it gradually reaches full rate:
  ```gdscript
  static func _calculate_currency_chance(
      base_chance: float,
      area_level: int,
      unlock_level: int,
      ramp_duration: int = 12
  ) -> float:
      if area_level < unlock_level:
          return 0.0
      var levels_since_unlock: int = area_level - unlock_level
      if levels_since_unlock >= ramp_duration:
          return base_chance
      var ramp_progress: float = float(levels_since_unlock) / float(ramp_duration)
      # Square-root curve: starts at ~30% at level 1, reaches 100% at level 12
      # This is the "immediate but low" shape from user decisions
      var ramp_multiplier: float = sqrt(ramp_progress)
      return base_chance * ramp_multiplier
  ```

  With sqrt: at unlock+1 = sqrt(1/12) ≈ 0.29x, at unlock+3 = sqrt(3/12) = 0.50x, at unlock+6 = sqrt(6/12) ≈ 0.71x, at unlock+12 = 1.0x. This is "immediate but low" — the player sees drops right away (29% rate) but it takes 12 levels to reach full.

  **Step C: Retune per-pack currency base chances**

  Remove the `area_multiplier` calculation entirely from `roll_pack_currency_drop()`. The old logarithmic area scaling (1.0x to 1.85x) was designed for 300-level ranges. With 25-level biomes, there's no need for area-based scaling — new currencies are introduced at each biome boundary instead.

  Update `pack_currency_rules` base chances to hit the ~1 per 3-5 packs target:
  ```gdscript

      "tack": {"chance": 0.25, "max_qty": 2},     # ~1 per 4 packs, sometimes 2

      "grand": {"chance": 0.20, "max_qty": 1},    # ~1 per 5 packs
      "claw": {"chance": 0.20, "max_qty": 1},     # ~1 per 5 packs
      "tuning": {"chance": 0.20, "max_qty": 1},   # ~1 per 5 packs

  ```

  Runic/Tack are slightly more generous (25% + max 2 qty) since they're the basic currencies. Advanced currencies are 20% (1 per 5 packs) since the player accumulates multiple currency types simultaneously at higher biomes.

  Remove the `area_multiplier` variable and its usage. The `effective_chance` calculation simplifies to:

  Add a new static function `roll_pack_item_drop()` that returns true/false for whether an item drops from this pack kill. The target is 1-3 items per map with 8-15 packs per map (avg ~11).

  For a target of ~2 items per 11 packs: probability ≈ 2/11 ≈ 0.18 per pack.
  This gives:
  - 8-pack map: ~1.5 items (expected)
  - 11-pack map: ~2.0 items (expected)
  - 15-pack map: ~2.7 items (expected)

  Which falls in the 1-3 range. The drop rate is constant (no scaling with area — per user decision).

  ```gdscript
  ## Per-pack item drop chance. Constant across all areas.
  ## Targets 1-3 items per map (8-15 packs per map).
  const PACK_ITEM_DROP_CHANCE: float = 0.18

  ## Rolls whether a pack kill drops an item.
  ## Returns true if an item should drop, false otherwise.
  ## Drop rate is constant across all biomes (no area scaling).
  static func roll_pack_item_drop() -> bool:
      return randf() < PACK_ITEM_DROP_CHANCE
  ```
