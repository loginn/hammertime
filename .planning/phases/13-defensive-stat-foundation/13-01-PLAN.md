  <name>Task 1: Create DefenseCalculator with full damage mitigation pipeline</name>
  <files>models/stats/defense_calculator.gd</files>
  <action>
Create `models/stats/defense_calculator.gd` as a static utility class following the StatCalculator pattern (class_name DefenseCalculator extends RefCounted, all static methods).

Implement these static methods:

1. `calculate_armor_reduction(armor: int, raw_physical_damage: float) -> float`
   - Formula: `float(armor) / (float(armor) + 5.0 * raw_physical_damage)`
   - Returns damage reduction percentage (0.0 to ~0.9)
   - Returns 0.0 if armor <= 0 or raw_physical_damage <= 0.0
   - No hard cap (diminishing returns naturally prevent reaching 100%)

2. `calculate_dodge_chance(evasion: int) -> float`
   - Formula: `float(evasion) / (float(evasion) + 200.0)`
   - Returns dodge chance (0.0 to 0.75)
   - Cap at 75%: `minf(raw_chance, 0.75)`
   - Returns 0.0 if evasion <= 0

3. `calculate_resistance_reduction(resistance: int) -> float`
   - Effective resistance clamped: `mini(resistance, 75)`
   - Returns: `float(effective) / 100.0` (0.0 to 0.75)
   - Raw resistance value can exceed 75 (over-capping allowed in stats, effective clamped at application)

4. `apply_es_split(mitigated_damage: float, current_es: float) -> Dictionary`
   - Split 50/50: es_portion = mitigated_damage * 0.5, life_portion = mitigated_damage * 0.5
   - If ES can't absorb its portion (es_portion > current_es): overflow to life
   - Returns: `{ "es_damage": float, "life_damage": float }`
   - If current_es <= 0.0: all damage goes to life (no split)

5. `calculate_damage_taken(raw_damage, damage_type, is_spell, hero_armor, hero_evasion, hero_energy_shield, hero_fire_res, hero_cold_res, hero_lightning_res, current_es) -> Dictionary`
   - Full pipeline in order: Evasion -> Resistances -> Armor -> ES/Life split
   - Stage 1 (Evasion): Only if NOT spell and evasion > 0. Roll randf() < dodge_chance. If dodged, return { "dodged": true, "life_damage": 0.0, "es_damage": 0.0 }
   - Stage 2 (Resistances): Only for elemental damage_type ("fire", "cold", "lightning"). Apply matching resistance reduction
   - Stage 3 (Armor): Only for damage_type == "physical" and armor > 0. Apply armor reduction using post-resistance damage as raw_damage input
   - Stage 4 (ES/Life split): If current_es > 0 and hero_energy_shield > 0, call apply_es_split. Otherwise all to life
   - Clamp damage to maxf(0.0, damage) before ES split
   - Returns: `{ "dodged": bool, "life_damage": float, "es_damage": float }`

Parameter types for calculate_damage_taken:
- raw_damage: float
- damage_type: String (one of "physical", "fire", "cold", "lightning")
- is_spell: bool
- hero_armor: int
- hero_evasion: int
- hero_energy_shield: int (max ES, used to check if hero has ES at all)
- hero_fire_res: int
- hero_cold_res: int
- hero_lightning_res: int
- current_es: float (current ES amount, mutable)

Add GDDoc comments to each method explaining the formula, like StatCalculator does. Include practical examples in comments (e.g., "100 armor vs 10 damage = 66.7% reduction").

Do NOT add any print statements -- this is a pure calculation utility like StatCalculator.

<task type="auto">
  <name>Task 2: Add energy shield tracking and damage application to Hero</name>
  <files>models/hero.gd</files>
  <action>
Modify `models/hero.gd` to add energy shield tracking and a new damage application method:

1. Add new property after `total_lightning_resistance`:
   ```
   var current_energy_shield: float = 0.0
   ```

2. Add `apply_damage(life_damage: float, es_damage: float) -> void` method:
   - Subtract es_damage from current_energy_shield: `current_energy_shield = maxf(0.0, current_energy_shield - es_damage)`
   - Subtract life_damage from health: `health -= life_damage`
   - Clamp health: `health = maxf(0.0, health)`
   - If health <= 0: call die()
   - No print statements in the hot path (combat tick will call this frequently)

3. Add `recharge_energy_shield() -> void` method:
   - Recharge 33% of max ES: `var recharge_amount := float(total_energy_shield) * 0.33`
   - Apply: `current_energy_shield = minf(current_energy_shield + recharge_amount, float(total_energy_shield))`
   - This is called between pack fights (Phase 15 combat loop)

4. Add `get_current_energy_shield() -> float` getter method:
   - Returns current_energy_shield

5. Update `update_stats() -> void`:
   - After `calculate_defense()` is called, set `current_energy_shield = float(total_energy_shield)`
   - This ensures ES resets to max when gear changes (player always has full ES after equipping)
   - Add this line after the existing `calculate_defense()` call

6. Update `revive() -> void`:
   - Add: `current_energy_shield = float(total_energy_shield)` after health reset
   - Hero revives with full ES

Do NOT modify the existing `take_damage()` method -- it remains for backward compatibility. The new `apply_damage()` method is the defense-aware replacement that gameplay_view will switch to in Plan 02.

Do NOT modify `calculate_defense()` -- it already correctly sums all defense stats from equipment.
