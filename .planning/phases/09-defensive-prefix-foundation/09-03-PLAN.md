  <name>Task 2: Remove hardcoded base stats from armor items and add stat_types to implicits</name>
  <files>models/items/basic_armor.gd, models/items/basic_helmet.gd, models/items/basic_boots.gd, models/items/light_sword.gd</files>
  <action>
The core issue: armor items have TWO sources of base stats -- a hardcoded `original_base_armor` property AND an implicit affix. The implicit has empty `stat_types=[]` so its value never enters StatCalculator. User direction: implicits should be the SOLE source of base stats.

**File: models/items/basic_armor.gd**

1. Set `original_base_armor = 0` (was 15) and `base_armor = 0` (was 15).
2. Add `[Tag.StatType.FLAT_ARMOR]` as the 6th argument to the Implicit constructor:
```gdscript
self.implicit = Implicit.new("Armor", Affix.AffixType.IMPLICIT, 3, 8, [Tag.ARMOR, Tag.DEFENSE], [Tag.StatType.FLAT_ARMOR])
```

The implicit with tier_range default (1,8) and base_min=3, base_max=8 at tier 8 produces: min=3*(8+1-8)=3, max=8*(8+1-8)=8. So BasicArmor will have base armor of 3-8 from the implicit alone. This is lower than the old hardcoded 15, but it scales with tier (a tier 1 implicit would give 24-64).

**File: models/items/basic_helmet.gd**

1. Set `original_base_armor = 0` (was 10) and `base_armor = 0` (was 10).
2. Add `[Tag.StatType.FLAT_ARMOR]` as the 6th argument:
```gdscript
self.implicit = Implicit.new("Armor", Affix.AffixType.IMPLICIT, 2, 5, [Tag.ARMOR, Tag.DEFENSE], [Tag.StatType.FLAT_ARMOR])
```

Tier 8 produces min=2, max=5 flat armor from implicit.

**File: models/items/basic_boots.gd**

1. Set `original_base_armor = 0` (was 8) and `base_armor = 0` (was 8).
2. Add `[Tag.StatType.MOVEMENT_SPEED]` as the 6th argument:
```gdscript
self.implicit = Implicit.new(
    "Movement Speed", Affix.AffixType.IMPLICIT, 1, 3, [Tag.SPEED, Tag.MOVEMENT], [Tag.StatType.MOVEMENT_SPEED]
)
```

Note: Boots implicit is Movement Speed, not Armor. The boots base armor was 8 but the implicit was Movement Speed -- this means boots will now have 0 base armor (their implicit provides movement speed instead). This is correct: boots get armor only from crafted affixes.

**File: models/items/light_sword.gd** (consistency fix)

The LightSword implicit "Attack Speed" also has empty stat_types, making it cosmetic. Add `[Tag.StatType.INCREASED_SPEED]` as the 6th argument for consistency:
```gdscript
self.implicit = Implicit.new(
    "Attack Speed", Affix.AffixType.IMPLICIT, 2, 5, [Tag.SPEED, Tag.ATTACK], [Tag.StatType.INCREASED_SPEED]
)
```

This makes the implicit attack speed value flow through StatCalculator.calculate_dps() where it will be picked up by the INCREASED_SPEED check (line 36-37 of stat_calculator.gd). The LightSword base_speed stays at 1 and the implicit adds a percentage speed bonus on top.

Do NOT change armor.gd, helmet.gd, or boots.gd -- their update_value() methods already correctly compute `original_base_X + calculate_flat_stat(all_affixes, FLAT_X)`. With original_base_X now 0, the flat total comes entirely from implicits and affixes. The percentage modifiers then apply on top. No calculation logic changes needed.

Do NOT change basic_ring.gd -- rings have no implicit stat_types issue (their implicit is Crit Chance which is handled separately in calculate_dps via CRIT_CHANCE stat_type... wait, actually check: BasicRing implicit has no stat_types either). Add `[Tag.StatType.CRIT_CHANCE]` to BasicRing implicit too:

**File: models/items/basic_ring.gd** (consistency fix)

```gdscript
self.implicit = Implicit.new(
    "Crit Chance", Affix.AffixType.IMPLICIT, 1, 2, [Tag.CRITICAL, Tag.ATTACK], [Tag.StatType.CRIT_CHANCE]
)
```

This makes the ring's implicit crit chance value flow through calculate_dps() where it will be picked up at line 42: `if Tag.StatType.CRIT_CHANCE in affix.stat_types`.
