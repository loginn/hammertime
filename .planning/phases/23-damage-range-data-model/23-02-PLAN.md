3. In `_init()` body, after existing field setup (the `self.value = randi_range(...)` line), add damage range template setup and initial roll:
```gdscript
# Store template bounds (scaled by tier, same as min_value/max_value)
self.dmg_min_lo = p_dmg_min_lo * (tier_range.y + 1 - tier)
self.dmg_min_hi = p_dmg_min_hi * (tier_range.y + 1 - tier)
self.dmg_max_lo = p_dmg_max_lo * (tier_range.y + 1 - tier)
self.dmg_max_hi = p_dmg_max_hi * (tier_range.y + 1 - tier)

# Roll initial damage range if this is a flat damage affix
if Tag.StatType.FLAT_DAMAGE in self.stat_types and (dmg_min_hi > 0 or dmg_max_hi > 0):
    self.add_min = randi_range(self.dmg_min_lo, self.dmg_min_hi)
    self.add_max = randi_range(self.dmg_max_lo, self.dmg_max_hi)
    # Guard: ensure add_min <= add_max
    if self.add_min > self.add_max:
        var tmp = self.add_min
        self.add_min = self.add_max
        self.add_max = tmp
```

4. Modify `reroll()` to handle flat damage affixes separately:
```gdscript
func reroll() -> void:
    if Tag.StatType.FLAT_DAMAGE in self.stat_types and (dmg_min_hi > 0 or dmg_max_hi > 0):
        # Damage range affix: re-roll from TEMPLATE bounds (never from rolled values)
        self.add_min = randi_range(self.dmg_min_lo, self.dmg_min_hi)
        self.add_max = randi_range(self.dmg_max_lo, self.dmg_max_hi)
        if self.add_min > self.add_max:
            var tmp = self.add_min
            self.add_min = self.add_max
            self.add_max = tmp
        print("reroll add_min=%d add_max=%d" % [add_min, add_max])
    else:
        # Non-damage affix: existing scalar reroll unchanged
        self.value = randi_range(self.min_value, self.max_value)
        print("reroll ", self.value)
```
CRITICAL: The reroll for damage affixes reads from `dmg_min_lo`/`dmg_min_hi` (immutable template bounds), NOT from `add_min`/`add_max` (mutable rolled values). This is what prevents range collapse on repeated Tuning Hammer applications.

8. Update `Affixes.from_affix()` to pass the four template bound fields:
```gdscript
static func from_affix(template: Affix) -> Affix:
    var affix_copy = Affix.new(
        template.affix_name,
        template.type,
        template.base_min,
        template.base_max,
        template.tags,
        template.stat_types,
        template.tier_range,
        template.dmg_min_lo,
        template.dmg_min_hi,
        template.dmg_max_lo,
        template.dmg_max_hi
    )
    return affix_copy
```
CRITICAL: Without this change, every item created from a template gets 0-0 damage range bounds. The `from_affix()` must pass through the template's bounds so the new Affix instance rolls its own add_min/add_max.

Wait -- there is a subtlety. The template's `dmg_min_lo` etc. are already tier-scaled (they were scaled in the template's own `_init()`). But `from_affix()` creates a NEW Affix via `_init()` which will re-scale them by tier. This would double-scale.

The fix: `from_affix()` should pass the template's BASE (unscaled) `p_dmg_*` values. But the template doesn't store the unscaled values separately -- it stores `dmg_min_lo` which is already `p_dmg_min_lo * tier_multiplier`.

Actually, looking at how `from_affix()` works: it passes `template.base_min` and `template.base_max` (the UNscaled p_min/p_max), and the new Affix._init() re-rolls a tier and re-computes `min_value`/`max_value` with its own tier. So `from_affix()` creates a fresh roll with a new random tier.

For the damage bounds, we need the same pattern: pass the UNSCALED base bounds. But the Affix class doesn't store the unscaled damage base bounds separately.

Solution: Store `base_dmg_min_lo` etc. (the unscaled values) similar to how `base_min`/`base_max` stores the unscaled p_min/p_max. BUT that adds 4 more fields. Simpler approach: the _init() already receives the unscaled values as parameters. The template stores `base_min`/`base_max` for unscaled values. We need the same for damage bounds.

REVISED APPROACH: Add 4 more storage fields for the base (unscaled) damage bounds, like `base_min`/`base_max` pattern:
```gdscript
var base_dmg_min_lo: int = 0
var base_dmg_min_hi: int = 0
var base_dmg_max_lo: int = 0
var base_dmg_max_hi: int = 0
```

In `_init()`, store the unscaled values:
```gdscript
self.base_dmg_min_lo = p_dmg_min_lo
self.base_dmg_min_hi = p_dmg_min_hi
self.base_dmg_max_lo = p_dmg_max_lo
self.base_dmg_max_hi = p_dmg_max_hi
```

Then `from_affix()` passes `template.base_dmg_min_lo` etc. to avoid double-scaling.

Update `to_dict()` and `from_dict()` to include the 4 base fields too (total: 10 new fields in serialization — 4 base bounds, 4 scaled bounds, 2 rolled results).

In `from_dict()`, after restoring `dmg_min_lo` etc., also restore:
