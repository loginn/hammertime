# Plan 01: Data Model + StatCalculator Spell Methods

Add base spell damage fields to Weapon and base_cast_speed to Ring, then create StatCalculator spell calculation methods that mirror the existing attack pipeline.

<tasks>

<task id="46-01-01">
<title>Add spell damage fields to Weapon</title>
<description>
Add three new fields to models/items/weapon.gd:
- `var base_spell_damage_min: int = 0`
- `var base_spell_damage_max: int = 0`
- `var base_cast_speed: float = 0.0`
- Computed getter `var base_spell_damage: int` returning average of min/max (mirrors base_damage)
- `var spell_dps: float = 0.0` for caching spell DPS

Update `update_value()` to also compute spell_dps using the new StatCalculator.calculate_spell_dps() method (added in task 46-01-03). If base_cast_speed == 0 and no INCREASED_CAST_SPEED affixes present, spell_dps stays 0.

All existing weapon subclasses inherit these defaults at 0, so no changes to STR/DEX weapon files needed.
</description>

<task id="46-01-03">
<title>Add calculate_spell_damage_range() and calculate_spell_dps() to StatCalculator</title>
<description>
Add two new static methods to models/stats/stat_calculator.gd:

1. `calculate_spell_damage_range(base_spell_min: int, base_spell_max: int, affixes: Array) -> Dictionary`
   - Mirrors calculate_damage_range() but uses FLAT_SPELL_DAMAGE instead of FLAT_DAMAGE
   - Uses INCREASED_SPELL_DAMAGE instead of INCREASED_DAMAGE for percentage scaling
   - Returns single "spell" element: {"spell": {"min": float, "max": float}}
   - Step 1: Start with base_spell_min/max as "spell" element
   - Step 2: Add flat spell damage from affixes with FLAT_SPELL_DAMAGE stat type
   - Step 3: Apply %increased spell damage (INCREASED_SPELL_DAMAGE, additive stacking)
   - Keep it simple: single "spell" element, no per-element split (deferred to Phase 47)

2. `calculate_spell_dps(base_spell_damage: float, base_cast_speed: float, affixes: Array, base_crit_chance: float = 5.0, base_crit_damage: float = 150.0) -> float`
   - Mirrors calculate_dps() but for spell channel
   - Step 1: Add flat spell damage from FLAT_SPELL_DAMAGE affixes
   - Step 2: Apply %increased spell damage (INCREASED_SPELL_DAMAGE)
   - Step 3: Apply cast speed: base_cast_speed * (1 + sum of INCREASED_CAST_SPEED / 100)
   - Step 4: Apply shared crit multiplier
   - Step 5: Return damage * speed * crit_multiplier
   - If base_cast_speed == 0 and no cast speed affixes, return 0.0 (no spell channel)

Note on cast speed aggregation: The cast speed comes from both weapon.base_cast_speed and ring.base_cast_speed. The Hero layer will sum these before calling calculate_spell_dps(). StatCalculator just takes the total base_cast_speed as input.
</description>
