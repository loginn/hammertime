# Plan 02: Hero Spell Stats + UI Display

Wire spell damage tracking through Hero and update ForgeView to display Attack DPS and Spell DPS separately with hide-zero logic.

<tasks>

<task id="46-02-01">
<title>Add spell damage tracking to Hero</title>
<description>
Add to models/hero.gd:

New fields:
- `var total_spell_dps: float = 0.0`
- `var spell_damage_ranges: Dictionary` initialized to `{"spell": {"min": 0.0, "max": 0.0}}`

New method `calculate_spell_damage_ranges() -> void`:
- Reset spell_damage_ranges to zero
- Collect all affixes from weapon (prefixes + suffixes + implicit) that have FLAT_SPELL_DAMAGE
- If weapon has base_spell_damage_min/max > 0, use those as base
- Add flat spell damage from ring implicit and affixes (Sapphire Ring's FLAT_SPELL_DAMAGE implicit)
- Apply %INCREASED_SPELL_DAMAGE from all equipment affixes
- Use StatCalculator.calculate_spell_damage_range() to compute ranges

New method `calculate_spell_dps() -> float`:
- Sum average spell damage from spell_damage_ranges
- Aggregate base_cast_speed from weapon + ring (both contribute)
- Collect INCREASED_CAST_SPEED affixes from weapon + ring
- Compute effective cast speed: (weapon.base_cast_speed + ring.base_cast_speed) * (1 + sum_cast_speed_pct/100)
- If effective cast speed == 0, set total_spell_dps = 0 and return
- Apply shared crit multiplier (total_crit_chance, total_crit_damage already calculated)
- Set total_spell_dps = avg_spell_damage * effective_cast_speed * crit_multiplier

Update `update_stats()` call order:
```
calculate_crit_stats()           # unchanged
calculate_damage_ranges()        # unchanged — attack ranges
calculate_spell_damage_ranges()  # NEW
calculate_dps()                  # unchanged — sets total_dps (attack DPS)
calculate_spell_dps()            # NEW
calculate_defense()              # unchanged
```

Add getter `get_total_spell_dps() -> float`.

Note: total_dps is NOT renamed in the data model (too many references). The UI layer handles the "Attack DPS" label rename.
</description>
