    **FIX 2 — Bench-occupied toast (forge_view.gd):**
    Gap reason: btn.disabled=true when bench occupied prevents pressed signal from firing, making the toast guard dead code.

    In _update_stash_display(), change line 394 from:
      `btn.disabled = (GameState.crafting_bench != null)`
    to:
      `btn.disabled = false`

    This keeps filled stash slot buttons always enabled (clickable). The bench-occupied guard in _on_stash_slot_pressed (line 408-410) will now fire and show the "Melt or equip first" toast.

    **FIX 3 — Animation bugs (forge_view.gd):**

    **3a — Yellow flash invisible:**
    In _on_stash_slot_pressed, move the _flash_stash_slot call to AFTER _update_stash_display. Current order (lines 424-428):
    ```
    _flash_stash_slot(slot_type, index)    # line 424
    update_current_item()                  # line 427
    _update_stash_display()                # line 428
    ```
    Change to:
    ```
    update_current_item()
    _update_stash_display()
    _flash_stash_slot(slot_type, index)    # AFTER display update so tween overwrites grey
    ```
    Keep update_melt_equip_states() and update_inventory_display() after as before.

    **3b — Double pulse:**
    In _pulse_stash_slots, before creating each tween, explicitly set the button's modulate to Color(1,1,1,1) to prevent the disabled-to-enabled theme transition from creating a visual pop. Add this line before `var tween := create_tween()`:
    ```
    btn.modulate = Color(1.0, 1.0, 1.0, 1.0)
    ```
    Then add `await get_tree().process_frame` BEFORE the for-loop to let the theme transition settle before starting the tween. This ensures one clean pulse instead of a theme-pop + tween-pulse double.

    Actually, since _pulse_stash_slots is not async, use call_deferred instead: rename _pulse_stash_slots to _pulse_stash_slots_impl, create a new _pulse_stash_slots that calls `call_deferred("_pulse_stash_slots_impl")`. This defers the pulse to the next frame, after the disabled->enabled theme transition has completed. In _pulse_stash_slots_impl, keep the existing logic but add `btn.modulate = Color(1.0, 1.0, 1.0, 1.0)` before each tween creation to ensure a clean starting state.
