# Requirements: Hammertime

**Defined:** 2026-02-16
**Core Value:** The crafting loop must feel rewarding — finding items, using hammers to shape them, and equipping the result to push further into harder content.

## v1.2 Requirements

Requirements for pack-based mapping milestone. Each maps to roadmap phases.

### Monster Packs

- [x] **PACK-01**: Monster packs have HP, damage, and elemental damage type (physical/fire/cold/lightning)
- [x] **PACK-02**: Maps contain a random number of packs within a range per biome
- [x] **PACK-03**: Biomes have damage type distributions (Forest = mostly physical, Shadow Realm = mostly elemental)
- [x] **PACK-04**: Pack HP and damage scale with area level

### Combat

- [x] **COMBAT-01**: Hero fights monster packs sequentially in idle auto-combat
- [x] **COMBAT-02**: Hero attacks pack and pack attacks hero back each combat tick
- [x] **COMBAT-03**: Hero can die (HP reaches 0) ending the current map run
- [x] **COMBAT-04**: Death loses map progress but keeps currency earned from killed packs
- [x] **COMBAT-05**: Hero revives and can start a new map after death
- [x] **COMBAT-06**: Completing all packs in a map advances to the next map

### Defense

- [x] **DEF-01**: Armor reduces physical damage using diminishing returns formula
- [x] **DEF-02**: Evasion provides dodge chance with diminishing returns (capped at 75%)
- [x] **DEF-03**: Elemental resistances reduce elemental damage (capped at 75%)
- [x] **DEF-04**: Energy shield acts as buffer HP (absorbs damage before life)
- [x] **DEF-05**: Energy shield recharges a percentage of total ES between pack fights

### Drops

- [x] **DROP-01**: Monster packs drop currency when killed
- [x] **DROP-02**: Map completion drops items
- [x] **DROP-03**: Currency earned from packs is kept on hero death

### UI

- [x] **UI-01**: Gameplay view shows pack-based combat instead of time-based clearing
- [x] **UI-02**: Pack HP and hero HP visible during combat
- [x] **UI-03**: Pack progress shown (e.g., "3/7 packs cleared")

## Future Requirements

Deferred to future release. Tracked but not in current roadmap.

### Totem System

- **TOTEM-01**: Totem pieces drop from packs and maps
- **TOTEM-02**: Totem has slots for pieces with effects on maps
- **TOTEM-03**: Totem pieces improve loot but make packs stronger (risk/reward)
- **TOTEM-04**: Favor accumulates from killing packs
- **TOTEM-05**: Activate totem in burst early game, sustain endgame

### Combat Polish

- **POLISH-01**: Energy shield recharge rate modifiers via affixes
- **POLISH-02**: Evasion entropy system (pseudo-random, prevent streaks)
- **POLISH-03**: Elemental status effects (fire ignites, cold chills, lightning shocks)
- **POLISH-04**: Elemental damage preview before entering biome
- **POLISH-05**: Visible damage breakdown in combat log

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Real-time combat with player timing | Fundamentally conflicts with idle genre |
| Per-monster loot drops | Item explosion in idle game = inventory nightmare |
| 100% damage immunity | Removes challenge; 75% resistance cap is ARPG standard |
| Complex elemental interactions (fire melts ice, etc.) | Massive complexity for questionable value |
| Revive/retry mechanic with penalty | Encourages "throw bodies at it" gameplay |
| Totem system | Deferred to v1.3+ — builds on pack-based mapping |
| Save/load system | Not yet scoped |
