    Modify `autoloads/save_manager.gd` to add auto-save and event-driven saves:

    Add constants:
    - AUTO_SAVE_INTERVAL = 300.0 (5 minutes)

    Add variables:
    - var auto_save_timer: Timer
    - var _save_pending: bool = false (debounce flag)

    In _ready():
    - Create Timer node for auto-save:
      ```
      auto_save_timer = Timer.new()
      auto_save_timer.wait_time = AUTO_SAVE_INTERVAL
      auto_save_timer.one_shot = false
      auto_save_timer.timeout.connect(_on_auto_save)
      add_child(auto_save_timer)
      auto_save_timer.start()
      ```
    - Connect event-driven save triggers:
      ```
      GameEvents.item_crafted.connect(_on_save_trigger)
      GameEvents.equipment_changed.connect(_on_equipment_save_trigger)
      GameEvents.area_cleared.connect(_on_area_save_trigger)
      ```

    Add debounced save trigger functions:
    - `func _on_auto_save() -> void`: Call _trigger_save()
    - `func _on_save_trigger(_arg = null) -> void`: Call _trigger_save()
    - `func _on_equipment_save_trigger(_slot: String, _item: Item) -> void`: Call _trigger_save()
    - `func _on_area_save_trigger(_level: int) -> void`: Call _trigger_save()
    - `func _trigger_save() -> void`:
      - If _save_pending, return (debounce — prevents rapid-fire saves from multiple signals in same frame)
      - Set _save_pending = true
      - Call call_deferred("_deferred_save") — saves AFTER current frame completes to avoid mid-operation inconsistency
    - `func _deferred_save() -> void`:
      - Set _save_pending = false
      - Call save_game()
      - If success: GameEvents.save_completed.emit()
      - If failure: GameEvents.save_failed.emit()

    The signal connections use the existing GameEvents signals:
    - item_crafted(item: Item) — fires when crafting view finishes an item
    - equipment_changed(slot: String, item: Item) — fires when hero equips/unequips
    - area_cleared(area_level: int) — fires when map is completed

    NOTE: item_crafted signal exists on GameEvents but check if it's emitted anywhere. If crafting_view doesn't emit it through GameEvents, add the emission in crafting_view.gd's finish_item() function: `GameEvents.item_crafted.emit(finished_item)` after the item is finished.

    **Save Toast (scenes/save_toast.tscn + save_toast.gd):**

    Create a simple Label-based scene:
    - Root node: Label (named "SaveToast")
    - Script: save_toast.gd
    - Position: Top-right corner of screen (anchor right, margin from edge). Use anchor_right = 1.0, anchor_top = 0.0, offset_left = -120, offset_top = 10, offset_right = -10, offset_bottom = 40.
    - Text: "Saved" (default)
    - Font size: 14 (small, unobtrusive)
    - Font color: white with slight transparency
    - Visible: false by default
    - Horizontal alignment: right

    save_toast.gd:
    ```
    extends Label

    func _ready() -> void:
        visible = false
        GameEvents.save_completed.connect(_on_save_completed)
        GameEvents.save_failed.connect(_on_save_failed)

    func _on_save_completed() -> void:
        show_toast("Saved")

    func _on_save_failed() -> void:
        show_toast("Save failed")

    func show_toast(message: String) -> void:
        text = message
        modulate.a = 1.0
        visible = true
        # Kill any existing tween
        var existing_tweens = get_tree().get_processed_tweens()  # Not available — just create new tween
        var tween := create_tween()
        tween.tween_interval(1.0)  # Hold visible for 1 second
        tween.tween_property(self, "modulate:a", 0.0, 0.5)  # Fade out over 0.5 seconds
        tween.tween_callback(func(): visible = false)
    ```

    **Settings Menu (scenes/settings_menu.tscn + settings_menu.gd):**

    Create a Panel-based scene for the settings overlay:
    - Root node: PanelContainer (named "SettingsMenu")
    - Layout: Centered overlay, ~300x200 pixels
    - Children:
      - VBoxContainer
        - Label: "Settings" (title, bold/large)
        - Button: "Save Game" (id: save_button)
        - Button: "New Game" (id: new_game_button) — this button's text changes for double confirmation
        - Button: "Close" (id: close_button)
    - Visible: false by default

    settings_menu.gd:
    ```
    extends PanelContainer

    signal menu_closed()
    signal new_game_started()

    @onready var save_button: Button = $VBoxContainer/SaveButton
    @onready var new_game_button: Button = $VBoxContainer/NewGameButton
    @onready var close_button: Button = $VBoxContainer/CloseButton

    var _new_game_confirming: bool = false

    func _ready() -> void:
        visible = false
        save_button.pressed.connect(_on_save_pressed)
        new_game_button.pressed.connect(_on_new_game_pressed)
        close_button.pressed.connect(_on_close_pressed)

    func open_menu() -> void:
        _reset_new_game_state()
        visible = true

    func _on_save_pressed() -> void:
        SaveManager.save_game()
        # Toast handles feedback via GameEvents.save_completed

    func _on_new_game_pressed() -> void:
        if not _new_game_confirming:
            # First click: change button to confirm
            _new_game_confirming = true
            new_game_button.text = "Are you sure?"
        else:
            # Second click: wipe and restart
            _new_game_confirming = false
            new_game_button.text = "New Game"
            SaveManager.delete_save()
            GameState.initialize_fresh_game()
            SaveManager.save_game()  # Save the fresh state immediately
            new_game_started.emit()
            visible = false

    func _on_close_pressed() -> void:
        _reset_new_game_state()
        visible = false
        menu_closed.emit()

    func _reset_new_game_state() -> void:
        _new_game_confirming = false
        new_game_button.text = "New Game"
    ```

    **Startup Flow (autoloads/game_state.gd):**

    The startup flow in GameState._ready() should be:
    1. Call initialize_fresh_game() (sets defaults)
    2. Attempt var loaded = SaveManager.load_game()
    3. If loaded == false AND SaveManager.has_save():
       - Save file exists but couldn't be loaded (corrupted)
       - Set a flag: var save_was_corrupted: bool = false
       - Set save_was_corrupted = true
       - (Toast will check this flag when scene is ready and show "Save could not be loaded")
    4. If loaded == true: state is restored, nothing else to do
    5. If loaded == false AND NOT SaveManager.has_save(): first launch, fresh game starts
    6. Apply debug_hammers override if flag is true (AFTER load, so debug mode always gets hammers)

    For the corrupted save toast, the save_toast needs to check GameState.save_was_corrupted on _ready() and show the warning. Add to save_toast.gd _ready():
    ```

        show_toast("Save could not be loaded")
        GameState.save_was_corrupted = false
    ```
