  <action>
    Add `to_dict() -> Dictionary` and `static func from_dict(data: Dictionary) -> Affix` to `affix.gd`:
    - to_dict() returns: affix_name, type (int), value, tier, tags, stat_types, tier_range_x, tier_range_y, base_min, base_max, min_value, max_value
    - from_dict() creates new Affix with template params, then OVERWRITES randomized fields (tier, value, min_value, max_value) with saved values. This is critical because Affix._init() randomizes tier and value.

    Add `to_dict()` and `from_dict()` to `implicit.gd` — Implicit extends Affix, so it can delegate to parent. The from_dict should return an Implicit (not Affix).

    Add `to_dict() -> Dictionary` to `item.gd` base class:
    - Returns: item_type (string like "LightSword"), item_name, tier, rarity (int), valid_tags, implicit (via implicit.to_dict()), prefixes (array of affix dicts), suffixes (array of affix dicts)
    - Add `func get_item_type_string() -> String` that returns "" in base (overridden in subclasses)

    Add `static func create_from_dict(data: Dictionary) -> Item` to `item.gd`:
    - Uses ITEM_TYPES const dictionary mapping type strings to constructors: {"LightSword": LightSword, "BasicHelmet": BasicHelmet, "BasicArmor": BasicArmor, "BasicBoots": BasicBoots, "BasicRing": BasicRing}
    - Creates instance via ITEM_TYPES[item_type].new()
    - Restores rarity from data
    - Clears default implicit/prefixes/suffixes, restores from saved data using Affix.from_dict() for prefixes/suffixes and Implicit.from_dict() for implicit
    - Calls item.update_value() after restoring all affixes to recalculate derived stats
    - Returns null with push_warning if item_type unknown

    Override `get_item_type_string()` in each concrete subclass:
    - LightSword -> "LightSword"
    - BasicArmor -> "BasicArmor"
    - BasicHelmet -> "BasicHelmet"
    - BasicBoots -> "BasicBoots"
    - BasicRing -> "BasicRing"

    Weapon, Armor, Helmet, Boots, Ring intermediate classes do NOT need overrides (they're abstract — never instantiated directly from saves).

    IMPORTANT: Do NOT save derived stats (dps, total_defense, base_armor after calculation, etc.). These are recalculated by update_value() on load. Only save the base template values that the _init() constructor sets, plus rarity and affixes.
  </action>

  <action>
    Create `autoloads/save_manager.gd` extending Node:

    Constants:
    - SAVE_PATH = "user://hammertime_save.json"
    - SAVE_VERSION = 1

    `func save_game() -> bool`:
    - Build save_data Dictionary with:
      - "version": SAVE_VERSION
      - "timestamp": Time.get_unix_time_from_system()
      - "hero_equipment": Dictionary mapping slot name to item.to_dict() (or null if empty). Slots: weapon, helmet, armor, boots, ring.
      - "currencies": GameState.currency_counts.duplicate()
      - "crafting_inventory": Dictionary mapping type name to item.to_dict() (or null). Get from GameState.crafting_inventory (see below).
      - "crafting_bench_item": The current_item on the crafting bench, serialized via to_dict() (or null). Get from GameState.crafting_bench_item.
      - "crafting_bench_type": The selected_item_type string from crafting. Get from GameState.crafting_bench_type.
      - "max_unlocked_level": int from GameState.max_unlocked_level
      - "area_level": int from GameState.area_level
    - Stringify with JSON.stringify(save_data, "\t") for readable format
    - Open file with FileAccess.open(SAVE_PATH, FileAccess.WRITE)
    - Check file != null, return false on failure with push_warning
    - Write with file.store_string(json_string)
    - Return true on success

    `func load_game() -> bool`:
    - Check FileAccess.file_exists(SAVE_PATH), return false if not
    - Open with FileAccess.open(SAVE_PATH, FileAccess.READ)
    - Check file != null, return false on failure
    - Read with file.get_as_text()
    - Parse with JSON.parse_string(), return false if null or not Dictionary
    - Call _migrate_save(parsed) for version migration
    - Restore hero equipment: for each slot in parsed["hero_equipment"], call Item.create_from_dict() and equip via GameState.hero.equip_item()
    - Restore currencies: GameState.currency_counts = parsed["currencies"]
    - Restore crafting inventory: GameState.crafting_inventory = deserialized items dict
    - Restore crafting bench: GameState.crafting_bench_item and crafting_bench_type
    - Restore area progress: GameState.max_unlocked_level and area_level
    - Call GameState.hero.update_stats() to recalculate all derived stats
    - Return true

    `func _migrate_save(data: Dictionary) -> Dictionary`:
    - Get saved_version from data.get("version", 1)
    - Empty migrations array for now (future-proofing)
    - Return data unchanged for version 1
    - Log migration if version changed

    `func delete_save() -> void`:
    - Delete save file with DirAccess.remove_absolute() if exists
    - Used by New Game flow

    `func has_save() -> bool`:
    - Return FileAccess.file_exists(SAVE_PATH)

    Modify `autoloads/game_state.gd`:
    - Move crafting inventory state from crafting_view.gd to game_state.gd for centralized persistence:
      - Add `var crafting_inventory: Dictionary = {}` — maps type string to Item or null

      - Add `var crafting_bench_type: String = "weapon"` — selected item type on bench
      - Add `var max_unlocked_level: int = 1`

      - Initialize empty crafting_inventory for each type
      - Set max_unlocked_level = 1, area_level = 1
      - Set crafting_bench_item = null, crafting_bench_type = "weapon"
    - Modify `_ready()`:
      - Call initialize_fresh_game() first
      - Then attempt SaveManager.load_game()
      - If load returns false AND SaveManager.has_save(), it means save is corrupted — will be handled by toast in Plan 02
      - If load returns true, state is restored
      - Keep debug_hammers logic (applies AFTER load if flag is true)

    Register SaveManager as autoload in project.godot:

    - Line: SaveManager="*res://autoloads/save_manager.gd"

The crafting_view will need to be updated to use GameState for inventory — but keep the UI wiring in crafting_view. Similarly, combat_engine reads area_level from GameState instead of its own var. These view-layer changes are minimal: replace `crafting_inventory` references with `GameState.crafting_inventory`, `area_level` with `GameState.area_level`, etc. Do this in crafting_view.gd, gameplay_view.gd, and combat_engine.gd.
