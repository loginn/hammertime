  **1. Restore rarity color in update_item_stats_display().**

  In the `update_item_stats_display()` function (around line 451), after each branch that sets `item_stats_label.text`, also set the label modulate to the item's rarity color:

  ```gdscript
  func update_item_stats_display() -> void:
      if item_stats_label == null:
          return

      if finished_item != null:
          item_stats_label.text = "Finished Item:\n\n" + get_item_stats_text(finished_item)
          item_stats_label.modulate = finished_item.get_rarity_color()
      elif current_item != null:
          item_stats_label.text = "Crafting:\n\n" + get_item_stats_text(current_item)
          item_stats_label.modulate = current_item.get_rarity_color()
      else:
          item_stats_label.text = "Item Stats:\n\nNo item on crafting bench"
          item_stats_label.modulate = Color.WHITE
  ```

  This restores the rarity color behavior that was in the old crafting_view.gd (line 117: `item_label.modulate = current_item.get_rarity_color()`). The `get_rarity_color()` method already exists on Item and returns WHITE for Normal, #6888F5 for Magic, #FFD700 for Rare.
