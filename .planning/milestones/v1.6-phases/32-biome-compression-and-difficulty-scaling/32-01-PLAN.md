<task type="auto">
  <name>Task 2: Redesign difficulty scaling with boss wall / relief / ramp-back curve</name>
  <files>models/monsters/pack_generator.gd</files>
  <action>
Redesign `get_level_multiplier()` in pack_generator.gd to implement the full difficulty curve.

**Step 1: Update the base growth rate constant.**
Change `GROWTH_RATE: float = 0.06` to `GROWTH_RATE: float = 0.10` (10% compounding per level).

**Step 2: Add biome boundary constants** near the top of the file:
```gdscript
const BIOME_BOUNDARIES: Array[int] = [25, 50, 75]  # Dark Forest, Cursed Woods, Shadow Realm start levels
```

**Step 3: Replace `get_level_multiplier()` with the new curve-driven implementation.**

The new function takes `area_level: int` and returns `float`. It must implement:

**A) Base multiplier** — Standard 10% compounding:
```
base = pow(1.0 + GROWTH_RATE, level - 1)
```

**B) Boss wall bonus** — Extra difficulty spike on the last 3 levels of each biome (levels 22-24, 47-49, 72-74). The spike is EXTRA on top of the base 10%:
- 3rd-to-last level: +15% extra multiplier
- 2nd-to-last level: +35% extra multiplier
- Last level: +60% extra multiplier

Implementation: For each biome boundary in BIOME_BOUNDARIES, check if `level` is within 3 levels below the boundary (i.e., `boundary - 3 <= level < boundary`). If so, apply the boss wall bonus:
```gdscript
var distance_from_boundary: int = boundary - level  # 3, 2, or 1
var boss_bonus: float
match distance_from_boundary:
    3: boss_bonus = 0.15
    2: boss_bonus = 0.35
    1: boss_bonus = 0.60
    _: boss_bonus = 0.0
return base * (1.0 + boss_bonus)
```

**C) Relief dip at biome starts** — The first level of each new biome (25, 50, 75) is a noticeable DROP below the boss wall peak. The dip must also account for the fact that new biome monsters have higher base stats (biome_config base stats jump at each boundary).

Calculate the relief multiplier so that the effective difficulty (multiplier * average_base_stats_of_new_biome) is roughly 70% of the boss wall peak effective difficulty (peak_multiplier * average_base_stats_of_old_biome). This creates genuine relief despite stronger monsters.

To keep this self-contained and not require reading biome_config at runtime, use pre-computed base stat ratios between biomes. Calculate these from the existing monster type base stats in biome_config:
- Forest avg HP: (36 + 24 + 30 + 15 + 12 + 45) / 6 = 27.0
- Dark Forest avg HP: (42.5 + 80 + 17.5 + 45 + 35) / 5 = 44.0
- Cursed Woods avg HP: (22.5 + 85 + 37.5 + 45 + 20) / 5 = 42.0
- Shadow Realm avg HP: (47.5 + 100 + 25 + 40 + 65 + 17.5) / 6 = 49.17

Store as a constant dictionary mapping boundary level to the stat ratio (new biome avg / old biome avg):
```gdscript
# Pre-computed avg base HP ratios: new_biome / old_biome
# Used to ensure relief dip accounts for base stat jumps
const BIOME_STAT_RATIOS: Dictionary = {
    25: 1.63,   # Dark Forest (44.0) / Forest (27.0)
    50: 0.955,  # Cursed Woods (42.0) / Dark Forest (44.0)
    75: 1.17,   # Shadow Realm (49.17) / Cursed Woods (42.0)
}
```

Relief formula: At the boundary level (e.g., level 25), compute the boss wall peak (level 24's effective multiplier), then set the relief multiplier so that:
```
relief_mult * stat_ratio = peak_effective * 0.70
relief_mult = (peak_base * (1.0 + 0.60)) * 0.70 / stat_ratio
```
Where `peak_base = pow(1.1, boundary - 2)` (the base multiplier at the last level before boundary).

**D) Ramp-back from relief** — After the relief dip, difficulty smoothly ramps back over 8 levels to rejoin the base curve. During levels boundary+1 through boundary+7, interpolate between the relief multiplier and the base multiplier using ease-in (quadratic):
```gdscript
var levels_into_biome: int = level - boundary  # 1 through 7
var t: float = float(levels_into_biome) / 8.0
var ease_t: float = t * t  # Quadratic ease-in (slow start, accelerating)
var ramp_mult: float = relief_value + (base - relief_value) * ease_t
return ramp_mult
```

After 8 levels (boundary+8 and beyond), just return the base multiplier — the curve has rejoined normal 10% compounding.

**E) Shadow Realm (75+)** — NO boss wall pattern repeats. Just smooth 10% compounding after the initial relief + ramp-back at levels 75-82. Level 83+ is pure `pow(1.1, level - 1)`.

**Step 4: Update the doc comment** on `get_level_multiplier()` to describe the full curve:
```gdscript

## Base: 10% compounding per level. Modified by:
## - Boss wall: +15/35/60% spike on the last 3 levels of each biome
## - Relief dip: first level of new biome drops to ~70% of boss wall peak (accounting for base stat jump)
## - Ramp-back: smooth quadratic ease over 8 levels to rejoin base curve
## - Shadow Realm (75+): smooth 10% compounding after initial ramp-back, no repeating boss walls
```

**Step 5: Update the doc comment** at the class level to reflect the new scaling:

**Step 6: Verify ordering** in the function — the check order should be:

2. Check if level is in boss wall zone (within 3 levels before boundary) → return boss wall multiplier
3. Check if level is in ramp-back zone (within 8 levels after boundary) → return ramp multiplier
4. Default → return base multiplier
