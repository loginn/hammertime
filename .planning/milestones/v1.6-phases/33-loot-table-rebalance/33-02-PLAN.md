  <name>Task 1: Fix hero health/armor double-counting, add global PERCENT_HEALTH, sync health on stat update</name>
  <files>models/hero.gd</files>
  <action>
  Fix three bugs in hero.gd's calculate_defense() and update_stats():

  **Bug 1: FLAT_HEALTH and FLAT_ARMOR double-counted on armor slots**

  The suffix loop at lines 207-228 iterates over ALL 5 equipment slots and adds FLAT_HEALTH/FLAT_ARMOR from suffixes. But armor/helmet/boots items already bake FLAT_HEALTH and FLAT_ARMOR into their base_health and base_armor via their update_value() methods (using StatCalculator.calculate_flat_stat). So for armor-slot items, these stats are counted TWICE: once in base_health/base_armor (from item), once in the suffix loop (from hero).

  Weapon and Ring do NOT bake FLAT_HEALTH or FLAT_ARMOR into any base property — their update_value() only handles damage stats. So the suffix loop IS needed for weapon/ring but NOT for armor slots.

  Fix: Split the suffix loop into two sections:
  1. For ALL 5 slots: process resistances (FIRE_RESISTANCE, COLD_RESISTANCE, LIGHTNING_RESISTANCE, ALL_RESISTANCE) — these are NOT baked into item base stats, so no double-counting risk.
  2. For ONLY weapon and ring slots: process FLAT_HEALTH and FLAT_ARMOR — these items do not bake them in.
  3. Do NOT process FLAT_HEALTH or FLAT_ARMOR for helmet, armor, or boots — already in base_health/base_armor.

  Concrete implementation — replace the current suffix loop (lines 207-228) with:

  ```gdscript
  # Add resistance from suffixes on ALL equipment slots
  # (resistances are never baked into item base stats)
  for slot in ["helmet", "armor", "boots", "weapon", "ring"]:
      if slot in equipped_items and equipped_items[slot] != null:
          var item = equipped_items[slot]
          if "suffixes" in item:
              for suffix in item.suffixes:
                  if Tag.StatType.FIRE_RESISTANCE in suffix.stat_types:
                      total_fire_resistance += suffix.value
                  if Tag.StatType.COLD_RESISTANCE in suffix.stat_types:
                      total_cold_resistance += suffix.value
                  if Tag.StatType.LIGHTNING_RESISTANCE in suffix.stat_types:
                      total_lightning_resistance += suffix.value
                  if Tag.StatType.ALL_RESISTANCE in suffix.stat_types:
                      total_fire_resistance += suffix.value
                      total_cold_resistance += suffix.value
                      total_lightning_resistance += suffix.value

  # Add FLAT_HEALTH and FLAT_ARMOR from weapon/ring suffixes ONLY.
  # Armor slots (helmet, armor, boots) already bake these into base_health/base_armor
  # via their update_value() -> StatCalculator.calculate_flat_stat() calls.
  for slot in ["weapon", "ring"]:
      if slot in equipped_items and equipped_items[slot] != null:
          var item = equipped_items[slot]
          if "suffixes" in item:
              for suffix in item.suffixes:
                  if Tag.StatType.FLAT_HEALTH in suffix.stat_types:
                      total_health += suffix.value
                  if Tag.StatType.FLAT_ARMOR in suffix.stat_types:
                      total_armor += suffix.value
  ```

  **Bug 2: Missing global PERCENT_HEALTH multiplier**

  After computing total_health (base 100 + all item base_health + weapon/ring FLAT_HEALTH), apply a global PERCENT_HEALTH pass. Collect PERCENT_HEALTH values from prefixes across ALL equipped items and apply them additively to total_health using StatCalculator.calculate_percentage_stat.

  Add this AFTER the weapon/ring FLAT_HEALTH/FLAT_ARMOR loop, BEFORE the max_health assignment:

  ```gdscript
  # Apply global PERCENT_HEALTH from prefixes on all equipment
  # Item-level update_value() applies PERCENT_HEALTH to individual item base_health,
  # but a global pass ensures the modifier scales the entire health pool.
  var all_percent_health_affixes: Array = []
  for slot in ["helmet", "armor", "boots", "weapon", "ring"]:
      if slot in equipped_items and equipped_items[slot] != null:
          var item = equipped_items[slot]
          if "prefixes" in item:
              all_percent_health_affixes.append_array(item.prefixes)
  total_health = int(StatCalculator.calculate_percentage_stat(
      float(total_health), all_percent_health_affixes, Tag.StatType.PERCENT_HEALTH
  ))
  ```

  IMPORTANT: This means PERCENT_HEALTH is applied TWICE for armor-slot items — once at item level (scaling that item's base_health) and once at hero level (scaling total pool). This is intentional and matches the PoE model where "% increased maximum life" scales the total pool. The item-level application is relatively small (1-3% of ~10-20 base health = negligible). The global pass is where the real scaling happens (1-3% of ~150+ total health = meaningful).

  **Bug 3: update_stats() does not sync health to max_health**

  In update_stats(), after calculate_defense() updates max_health, sync current health:

  ```gdscript
  func update_stats() -> void:
      """Recalculate all hero stats based on equipped items"""
      calculate_crit_stats()
      calculate_damage_ranges()
      calculate_dps()
      calculate_defense()
      current_energy_shield = float(total_energy_shield)
      # Sync health to new max_health after stat recalculation
      health = max_health
  ```

  This ensures that when gear is equipped/unequipped or on hero init, health always equals max_health. The hero starts combat at full health. During combat, take_damage/heal manage health independently. On revive, health is already set to max_health by revive().
