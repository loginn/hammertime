<task type="auto">
  <name>Task 1: Create MonsterType and MonsterPack Resource classes</name>
  <files>models/monsters/monster_type.gd, models/monsters/monster_pack.gd</files>
  <action>
Create `models/monsters/` directory.

Create `models/monsters/monster_type.gd`:
- `class_name MonsterType extends Resource`
- Properties:
  - `var type_name: String` -- display name (e.g., "Forest Bear", "Shadow Imp")
  - `var base_hp: float` -- base hit points before area level scaling
  - `var base_damage: float` -- base damage per hit before area level scaling
  - `var base_attack_speed: float` -- attacks per second (NOT scaled by area level, preserves type identity)
- Constructor `_init()` with defaults for all params (required for Godot resource loader compatibility -- same pattern as Affix._init())
- Static factory method `static func create(p_name: String, p_hp: float, p_dmg: float, p_speed: float) -> MonsterType` for clean construction

Create `models/monsters/monster_pack.gd`:
- `class_name MonsterPack extends Resource`
- Properties:
  - `var pack_name: String` -- name from monster type
  - `var hp: float` -- current HP (scaled, decreases during combat)
  - `var max_hp: float` -- maximum HP (scaled, set at creation)
  - `var damage: float` -- damage per hit (scaled)
  - `var attack_speed: float` -- attacks per second (from type, not scaled)
  - `var element: String` -- one of "physical", "fire", "cold", "lightning"
- Method `func is_alive() -> bool` -- returns hp > 0
- Method `func take_damage(amount: float) -> void` -- reduces hp, clamps at 0
- Method `func get_hp_percentage() -> float` -- returns hp / max_hp

Both files must have return type hints on all functions (project convention from Phase 1).

<task type="auto">
  <name>Task 2: Create BiomeConfig Resource with four biome definitions</name>
  <files>models/monsters/biome_config.gd</files>
  <action>
Create `models/monsters/biome_config.gd`:
- `class_name BiomeConfig extends Resource`
- Properties:
  - `var biome_name: String` -- display name (Forest, Dark Forest, etc.)
  - `var level_min: int` -- minimum area level for this biome
  - `var level_max: int` -- maximum area level (-1 for uncapped, Shadow Realm)
  - `var primary_element: String` -- dominant element for this biome
  - `var element_weights: Dictionary` -- maps element string to float weight (e.g., {"physical": 0.40, "fire": 0.20, "cold": 0.20, "lightning": 0.20})
  - `var monster_types: Array[MonsterType]` -- pool of named monster types for this biome

- Static constant `BIOMES: Array[BiomeConfig]` holding the four configured biomes. Build each biome in a static `_get_biomes()` method called lazily:

  **Forest (levels 1-99):**
  - Primary element: "physical"
  - Element weights: physical 0.40, fire 0.20, cold 0.20, lightning 0.20
  - Monster types (5-6 named types, natural beast theme):
    - Forest Bear: high base_hp (~120), medium base_damage (~12), slow attack_speed (~0.8)
    - Timber Wolf: medium base_hp (~80), medium base_damage (~10), medium attack_speed (~1.2)
    - Wild Boar: high base_hp (~100), high base_damage (~14), slow attack_speed (~0.9)
    - Venomous Spider: low base_hp (~50), medium base_damage (~11), fast attack_speed (~1.8)
    - Forest Sprite: low base_hp (~40), low base_damage (~8), fast attack_speed (~2.0)
    - Bramble Golem: very high base_hp (~150), low base_damage (~7), very slow attack_speed (~0.6)

  **Dark Forest (levels 100-199):**
  - Primary element: "fire"
  - Element weights: fire 0.40, physical 0.25, cold 0.20, lightning 0.15
  - Monster types (5-6 named types, corrupted/burning forest theme):
    - Ember Hound: medium base_hp (~85), high base_damage (~15), fast attack_speed (~1.5)
    - Charred Treant: very high base_hp (~160), medium base_damage (~11), very slow attack_speed (~0.5)
    - Flame Wisp: low base_hp (~35), high base_damage (~16), very fast attack_speed (~2.2)
    - Ash Stalker: medium base_hp (~90), medium base_damage (~13), medium attack_speed (~1.1)
    - Cinder Beetle: medium base_hp (~70), low base_damage (~9), fast attack_speed (~1.7)

  **Cursed Woods (levels 200-299):**
  - Primary element: "cold"
  - Element weights: cold 0.40, lightning 0.25, fire 0.20, physical 0.15
  - Monster types (5-6 named types, frozen/cursed theme):
    - Frost Wraith: low base_hp (~45), very high base_damage (~18), fast attack_speed (~1.6)
    - Frozen Troll: very high base_hp (~170), medium base_damage (~12), very slow attack_speed (~0.5)
    - Ice Crawler: medium base_hp (~75), medium base_damage (~14), medium attack_speed (~1.3)
    - Cursed Dryad: medium base_hp (~90), high base_damage (~15), slow attack_speed (~0.9)
    - Blizzard Imp: low base_hp (~40), medium base_damage (~11), very fast attack_speed (~2.3)

  **Shadow Realm (levels 300+, level_max = -1):**
  - Primary element: "lightning"
  - Element weights: lightning 0.40, fire 0.25, cold 0.25, physical 0.10
  - Monster types (5-6 named types, eldritch horror theme):
    - Void Stalker: medium base_hp (~95), very high base_damage (~20), fast attack_speed (~1.4)
    - Eldritch Horror: very high base_hp (~200), high base_damage (~16), very slow attack_speed (~0.4)
    - Storm Phantom: low base_hp (~50), very high base_damage (~19), very fast attack_speed (~2.0)
    - Shadow Fiend: medium base_hp (~80), high base_damage (~17), medium attack_speed (~1.2)
    - Abyssal Crawler: high base_hp (~130), medium base_damage (~13), slow attack_speed (~0.8)
    - Rift Elemental: low base_hp (~35), very high base_damage (~22), fast attack_speed (~1.8)

- Static method `static func get_biome_for_level(area_level: int) -> BiomeConfig` -- returns the biome config for a given area level (matches current biome boundaries: <100 Forest, <200 Dark Forest, <300 Cursed Woods, 300+ Shadow Realm)

Note: These base stat values are pre-scaling. They represent level 1 equivalents. The PackGenerator (Plan 02) will apply `pow(1 + growth_rate, area_level - 1)` to base_hp and base_damage. Attack speed is NOT scaled.

Note: Element weights use ~40% primary / ~60% distributed pattern per CONTEXT.md. Physical is NOT guaranteed in off-element mix (Shadow Realm has only 10% physical).
