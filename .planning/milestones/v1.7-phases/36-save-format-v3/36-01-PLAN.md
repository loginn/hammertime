<task type="auto">
  <name>Task 1: Upgrade save format to v3 with prestige field persistence and delete-on-old-version policy</name>
  <files>autoloads/save_manager.gd</files>
  <action>
In autoloads/save_manager.gd, make the following changes:

1. **Bump SAVE_VERSION from 2 to 3** — change `const SAVE_VERSION = 2` to `const SAVE_VERSION = 3`.

2. **Replace migration with delete-and-fresh-start in load_game()** — After parsing JSON and before calling _restore_state(), add a version check. If `saved_version < SAVE_VERSION`, call `delete_save()` and return false. Remove the `data = _migrate_save(data)` line. The updated load_game() should:
   - Parse JSON as before
   - Check `int(data.get("version", 1))` — if less than SAVE_VERSION, push_warning with the old version number, call delete_save(), return false
   - Otherwise proceed to `_restore_state(data)`

3. **Remove dead migration code** — Delete the `_migrate_save()` function and the `_migrate_v1_to_v2()` function entirely. They are unreachable since load_game() now deletes old saves before reaching them.

4. **Add prestige fields to _build_save_data()** — After the `"area_level"` key, add three new keys:
   - `"prestige_level": GameState.prestige_level`
   - `"max_item_tier_unlocked": GameState.max_item_tier_unlocked`
   - `"tag_currency_counts": GameState.tag_currency_counts.duplicate()`
   Use `.duplicate()` on tag_currency_counts to match the existing `currency_counts.duplicate()` pattern (shallow copy prevents live reference issues).

5. **Add prestige field restoration to _restore_state()** — After the area progress restoration block (after `GameState.area_level = ...`) and BEFORE `GameState.hero.update_stats()`, add:
   ```gdscript
   # Restore prestige state (v3)
   GameState.prestige_level = int(data.get("prestige_level", 0))
   GameState.max_item_tier_unlocked = int(data.get("max_item_tier_unlocked", 8))

   # Restore tag currencies (clear first to avoid stale keys from previous load)
   GameState.tag_currency_counts = {}
   var saved_tag_currencies: Dictionary = data.get("tag_currency_counts", {})
   for tag_type in saved_tag_currencies:
       GameState.tag_currency_counts[tag_type] = int(saved_tag_currencies[tag_type])
   ```
   IMPORTANT: Clear tag_currency_counts to `{}` BEFORE restoring to avoid stale keys persisting from a previous session. This is different from currency_counts which has fixed keys.
   Defaults: prestige_level=0, max_item_tier_unlocked=8 (P0 baseline), tag_currency_counts={}.
   No validation/clamping per user decision — trust the data.

6. **Update import_save_string()** — The existing import flow calls `_migrate_save()` which no longer exists. Replace the migration call in import_save_string():
   - Remove `data = _migrate_save(data)` line
   - Allow old-version import strings to succeed (v2 imports get default prestige values from _restore_state()'s .get() defaults). This is more user-friendly than rejection. The imported data gets saved as v3 via the `save_game()` call that already follows _restore_state().
   - Keep the existing `> SAVE_VERSION` newer-version rejection check unchanged.
  </action>

  <name>Task 2: Add prestige auto-save via prestige_completed signal</name>
  <files>autoloads/save_manager.gd</files>
  <action>
In autoloads/save_manager.gd, make two additions:

1. **Connect prestige_completed signal in _ready()** — After the existing `GameEvents.area_cleared.connect(_on_area_save_trigger)` line, add:
   ```gdscript
   GameEvents.prestige_completed.connect(_on_prestige_completed)
   ```

2. **Add the prestige handler method** — After the `_on_area_save_trigger` function, add:
   ```gdscript
   func _on_prestige_completed(_new_level: int) -> void:
       save_game()
   ```
   Use direct `save_game()` (NOT the debounced `_trigger_save()`). Prestige is a singular, high-stakes event — the save must complete immediately, not be deferred. The `_trigger_save()` debounce exists for rapid-fire events (crafting, area clear); prestige doesn't have that problem.

   The underscore prefix on `_new_level` suppresses the unused parameter warning while keeping the signal signature intact.

   **Timing correctness:** `prestige_completed` fires at the END of `execute_prestige()` in PrestigeManager, after prestige_level is advanced, max_item_tier_unlocked is updated, _wipe_run_state() has run, and _grant_random_tag_currency() has completed. The save captures the correct post-prestige snapshot.
  </action>

  </verify>
