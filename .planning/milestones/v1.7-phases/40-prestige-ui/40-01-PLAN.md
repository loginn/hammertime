<task id="2" title="Create prestige_view.gd script with unlock table and confirmation flow" depends_on="1">
Create `scenes/prestige_view.gd` with the full prestige view logic.

**Script structure:**

```gdscript
extends Node2D

signal prestige_triggered()

@onready var current_level_label: Label = $CurrentLevelLabel
@onready var next_info_label: Label = $NextInfoLabel
@onready var next_reward_label: Label = $NextRewardLabel
@onready var unlock_table: VBoxContainer = $UnlockTable
@onready var prestige_button: Button = $PrestigeButton

var prestige_confirm_pending: bool = false
var prestige_timer: Timer
```

**_ready() function:**
1. Connect `prestige_button.pressed` to `_on_prestige_pressed`
2. Create `prestige_timer = Timer.new()`, name="PrestigeTimer", one_shot=true, wait_time=3.0, connect timeout to `_on_prestige_timer_timeout`, add_child(prestige_timer)
3. Call `_update_display()`
4. Connect `GameEvents.currency_dropped` to `_on_currency_changed` (to update button disabled state dynamically)

**_update_display() function:**
1. Set `current_level_label.text = "Current Prestige: P" + str(GameState.prestige_level)`
2. If `GameState.prestige_level >= PrestigeManager.MAX_PRESTIGE_LEVEL`:
   - `next_info_label.text = "Max Prestige Reached"`
   - `next_reward_label.text = ""`
3. Else:
   - Get next level = `GameState.prestige_level + 1`
   - Get cost dict = `PrestigeManager.get_next_prestige_cost()`
   - Build cost string from dict. The key "forge" maps to display name "Forge Hammers". Format: `"Next: " + str(cost["forge"]) + " Forge Hammers"` (all current costs use forge only)
   - Get tier unlock = `PrestigeManager.ITEM_TIERS_BY_PRESTIGE[next_level]`
   - `next_info_label.text = cost_string`
   - Build reward text: `"Unlocks: Item Tier " + str(tier_unlock)`. If next_level == 1, append " + Tag Hammers". Set `next_reward_label.text`
4. Call `_build_unlock_table()`
5. Call `_update_button_state()`

**_build_unlock_table() function:**
1. Clear all children of `unlock_table` (free existing rows)
2. Loop `for level in range(1, PrestigeManager.MAX_PRESTIGE_LEVEL + 1):` (1 through 7)
3. For each level, create an HBoxContainer row, custom_minimum_size.y = 35
4. Determine row state:
   - `completed` if `level <= GameState.prestige_level`
   - `next` if `level == GameState.prestige_level + 1`
   - `future` if `level > GameState.prestige_level + 1`
5. Create 5 Labels as children of the row HBoxContainer, each with custom_minimum_size matching header widths (60, 80, 160, 200, 200), horizontal_alignment=1:
   - **Status** (60px): completed = "[checkmark symbol] " (use unicode checkmark), next = "> " (arrow), future = "[lock symbol]" (use unicode padlock or just "-")
   - **Level** (80px): "P" + str(level)
   - **Tier** (160px): "Tier " + str(PrestigeManager.ITEM_TIERS_BY_PRESTIGE[level])
   - **Reward** (200px): If level == 1: "Tag Hammers". Else: "-"
   - **Cost** (200px): If completed: str(PrestigeManager.PRESTIGE_COSTS[level]["forge"]) + " Forge". If next: str(PrestigeManager.PRESTIGE_COSTS[level]["forge"]) + " Forge". If future: "???"
6. Apply row coloring via modulate on the HBoxContainer:
   - completed: Color(0.4, 0.7, 0.4, 1) (green-tinted)
   - next: Color(1.0, 1.0, 1.0, 1) (bright white, full opacity)
   - future: Color(0.5, 0.5, 0.5, 1) (grey, locked feel)
7. Add the row HBoxContainer to `unlock_table`

**_update_button_state() function:**
```gdscript
func _update_button_state() -> void:
    if GameState.prestige_level >= PrestigeManager.MAX_PRESTIGE_LEVEL:
        prestige_button.disabled = true
        prestige_button.text = "Max Prestige"
    else:
        prestige_button.disabled = not PrestigeManager.can_prestige()
        if not prestige_confirm_pending:
            prestige_button.text = "Upgrade your forge"
```

**_on_prestige_pressed() function:**
Following forge_view.gd equip pattern (lines 482-519):
```gdscript
func _on_prestige_pressed() -> void:
    if not PrestigeManager.can_prestige():
        return
    if not prestige_confirm_pending:
        prestige_confirm_pending = true
        prestige_button.text = "Reset progress?"
        prestige_timer.start()
    else:
        prestige_confirm_pending = false
        prestige_timer.stop()
        _execute_prestige()
```

**_execute_prestige() function:**
Following settings_view.gd pattern (lines 29-41) — prestige_view does the action, emits signal for main_view to handle reload:
```gdscript
func _execute_prestige() -> void:
    PrestigeManager.execute_prestige()
    SaveManager.save_game()
    prestige_triggered.emit()
```

**_on_prestige_timer_timeout() function:**
```gdscript
func _on_prestige_timer_timeout() -> void:
    prestige_confirm_pending = false
    _update_button_state()
```

**_on_currency_changed(_drops: Dictionary) function:**
```gdscript
func _on_currency_changed(_drops: Dictionary) -> void:
    _update_button_state()
```

**reset_state() function** (called by main_view when navigating away, matching settings_view pattern):
```gdscript
func reset_state() -> void:
    prestige_confirm_pending = false
    prestige_timer.stop()
    _update_button_state()
```

**Acceptance:**

- Timer resets button text after 3 seconds if no second click
- P1 row shows "Tag Hammers" in reward column; other rows show "-"
- Future costs show "???"
- reset_state() clears confirmation pending state
