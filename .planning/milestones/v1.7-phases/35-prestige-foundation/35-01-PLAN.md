    **Create `autoloads/prestige_manager.gd`** — new file, `extends Node` with `class_name` omitted (autoloads don't need class_name since they're accessed by registered name).

    Constants at top:
    - `const MAX_PRESTIGE_LEVEL: int = 7`
    - `const PRESTIGE_COSTS: Dictionary` — keyed 1-7, each value is a Dictionary of `{currency_type: amount}`. P1 = `{"forge": 100}`. P2-P7 = `{"forge": 999999}` (stub values per user decision).
    - `const ITEM_TIERS_BY_PRESTIGE: Array[int] = [8, 7, 6, 5, 4, 3, 2, 1]` — index 0 = P0 baseline (tier 8 = lowest quality), index 7 = P7 (tier 1 = best quality). `ITEM_TIERS_BY_PRESTIGE[prestige_level]` works for all levels 0-7.
    - `const TAG_TYPES: Array[String] = ["fire", "cold", "lightning", "defense", "physical"]`

    Methods:
    - `func can_prestige() -> bool:` — Returns false if `prestige_level >= MAX_PRESTIGE_LEVEL`. Otherwise checks `PRESTIGE_COSTS[prestige_level + 1]` against `GameState.currency_counts` using `.get(currency_type, 0)`.
    - `func get_next_prestige_cost() -> Dictionary:` — Returns cost dict for next level, or empty dict if at max.
    - `func execute_prestige() -> bool:` — The full prestige sequence. CRITICAL ORDERING:
      1. Validate via `can_prestige()` — return false if not met
      2. Spend currencies BEFORE wipe — loop over cost dict, call `GameState.spend_currency(currency_type)` in a range loop for each amount
      3. Set `GameState.prestige_level = next_level`
      4. Set `GameState.max_item_tier_unlocked = ITEM_TIERS_BY_PRESTIGE[next_level]`
      5. Call `GameState._wipe_run_state()` — this resets run-scoped state
      6. Call `_grant_random_tag_currency()` — AFTER wipe so the granted currency isn't wiped
      7. Emit `GameEvents.prestige_completed.emit(next_level)`
      8. Return true
    - `func _grant_random_tag_currency() -> void:` — Picks a random tag from TAG_TYPES via `.pick_random()`, initializes key in `GameState.tag_currency_counts` if missing, increments by 1.

    See 35-RESEARCH.md "Code Examples > PrestigeManager autoload skeleton" for the complete implementation.

    **Modify `autoloads/game_state.gd`** — add 3 new fields and 1 new method:

    New fields (add after `area_level` declaration, before `save_was_corrupted`):
    ```gdscript
    # Prestige state — survives resets (NOT wiped by _wipe_run_state)
    var prestige_level: int = 0
    var max_item_tier_unlocked: int = 8  # P0 = tier 8 (lowest quality ceiling)

    # Tag currency inventory — separate from standard currency_counts
    # Wiped on prestige (run currency), but kept separate for Phase 39 gating
    var tag_currency_counts: Dictionary = {}
    ```

    Update `initialize_fresh_game()` — add these 3 lines AFTER the existing `area_level = 1` line:
    ```gdscript
    # Reset prestige state (only for truly fresh games)
    prestige_level = 0
    max_item_tier_unlocked = 8
    tag_currency_counts = {}
    ```

    New method `_wipe_run_state()` — add as a new function AFTER `initialize_fresh_game()`. This resets exactly 5 categories without touching prestige_level or max_item_tier_unlocked:
    1. Area progress: `area_level = 1`, `max_unlocked_level = 1`
    2. Hero: `hero = Hero.new()` then set all 5 equipment slots to null
    3. Crafting inventory: fresh empty arrays per slot, starter weapon `LightSword.new()` in weapon slot, `crafting_bench_type = "weapon"`
    4. Standard currencies: reset to fresh defaults `{"runic": 1, "forge": 0, "tack": 0, "grand": 0, "claw": 0, "tuning": 0}`
    5. Tag currencies: `tag_currency_counts = {}`
    6. Call `hero.update_stats()` at the end to ensure derived stats are consistent (matches _restore_state pattern in SaveManager)

    CRITICAL: Do NOT call `initialize_fresh_game()` from `_wipe_run_state()`. They are separate methods. `_wipe_run_state()` deliberately leaves `prestige_level` and `max_item_tier_unlocked` untouched.

    **Modify `autoloads/game_events.gd`** — add 2 new signals at the bottom:
